# 结合长期相关性与短期价格形态来捕捉alpha

- **链接**: 结合长期相关性与短期价格形态来捕捉alpha.md
- **作者**: 顾问 QP72475 (点塔王) (Rank 84)
- **发布时间/热度**: 4个月前, 得票: 34

## 帖子正文

```
Rank(correlation(delay((open - close), 1), close, 200)) + Rank((open - close))
```

1. **第一项：昨日价差与收盘价的相关性排名**
   - **`delay((open - close), 1)`** ：计算前一个交易日的K线实体长度（开盘价减收盘价）。若值为正，通常意味着收阴线；若为负，则为阳线。
   - **`correlation(..., close, 200)`** ：计算上述“昨日价差”与“当日收盘价”在过去200个交易日内的 **相关性** 。这是一个非常长周期的统计量。它试图衡量：在长期来看，前一天的阴线/阳线实体大小，与当天股价的高低是否存在某种稳定的同向或反向关系。
   - **`rank(...)`** ：对计算出的相关性值在截面上进行排序，将其转化为相对强弱排序分，使得因子在横截面上具有可比性。
2. **第二项：当日价差的排名**
   - **`(open - close)`** ：计算当日的K线实体长度（开盘价减收盘价）。
   - **`rank(...)`** ：同样对该值在截面上进行排序。
3. **因子合成**
   - 最终将上述两项排名直接 **相加** 。这意味着，该因子同时奖励那些在长期相关性中表现出某种模式（第一项排名高）且当日具有特定形态（第二项排名高）的股票。

### 量化实现步骤

基于上述拆解，可以将其量化实现流程清晰地分为以下几步。以下实现基于通用的面板数据格式，其中包含每种股票在不同时间点的 `open` （开盘价）、 `close` （收盘价）等字段  [-2](https://blog.csdn.net/qq_41996852/article/details/126728863) 。

#### 第一步：计算基础变量

对于每一只股票和每一天，计算两个基础列：

- `delay_diff`  = 前一天的  `(open - close)`  值。
- `today_diff`  = 当天的  `(open - close)`  值。

#### 第二步：计算长期相关性

对于每一只股票，使用一个长度为200天的滚动窗口，计算  `delay_diff`  与  `close`  的相关系数。这一步在时间序列上进行  [-2](https://blog.csdn.net/qq_41996852/article/details/126728863) 。

- `correlation_200`  =  `rolling_corr(delay_diff, close, window=200)`

#### 第三步：截面排名

在每个交易日，对所有股票执行截面排名操作，将前两步计算出的值转换为排名分  [-2](https://blog.csdn.net/qq_41996852/article/details/126728863) 。

- `rank_corr`  =  `rank(correlation_200)`
- `rank_diff`  =  `rank(today_diff)`

#### 第四步：因子合成

将两个排名分相加，得到最终的Alpha #037因子值。

- `alpha_037`  =  `rank_corr + rank_diff`

### 应用要点与风险提示

在将此因子应用于实际策略时，有几个关键点需要关注：

- **数据清洗** ：在计算相关系数前，需确保处理 `open` 和 `close` 价格中的缺失值或异常值，以避免计算出错。
- **中性化处理** ：原始的 `alpha_037` 值通常会包含市场风格的影响。在实际应用中，常常需要对其进行进一步的 **中性化处理** ，例如剔除行业（ `indclass` ）和市值（ `cap` ）等因素的影响，以得到纯粹的选股信号  [-2](https://blog.csdn.net/qq_41996852/article/details/126728863) 。
- **逻辑理解** ：该因子的逻辑较为抽象。第一项捕捉的是“昨日K线实体”与“今日收盘价水平”的长期稳定关系；第二项则是简单的“今日K线实体”大小。二者相加，可以理解为一种结合了长期统计规律和短期形态的复合判断。
