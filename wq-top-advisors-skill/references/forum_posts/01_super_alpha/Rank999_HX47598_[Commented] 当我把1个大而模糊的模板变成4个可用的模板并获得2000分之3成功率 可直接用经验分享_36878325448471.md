**0/ 引子**

**曾经我有一个出货率奇高，但是搜索空间巨大的模板：** 
（这个模板应该也是论坛上看来的，但是忘记出处了 :< )
"{ts_op1}({ts_op2}({x}, {d1}), {d2})"

> ts_op1 和 ts_op2 分别是两个ts operator的列表
> 它有以下两个约束条件
> - TemplateCondition.d1_larger_than_d2 - d1必须大于d2
> - TemplateCondition.ts_op1_neq_ts_op2 - ts_op1不能等于ts_op2

题外话：TemplateCondition.* 用于模板的条件约束，通过python class加staticmethod，如果有小伙伴感兴趣我可以单独写一个post分享

**1/ 质疑**

当然，敏锐的同学已经发现了，搜索空间巨大怎么能叫出货率高呢？

确实，在这里我说的出货率高，其实是”能找到信号“的确定性比较大。这个模板有两个致命问题：

- 1. 空间大而稀疏。ts_op1和ts_op2分别有20个操作符，加上d1和d2的自由度，搜索空间来到了平均一个字段`20 * 20 * 5 * 4` = 8000个表达式
- 2. 经济学意义不明确

过去几个月其实我得过且过，感觉只要能出货等一下也没关系（一般我就sample 10%然后等到出货足够就停止）；但是最近simulation限流，拖延症患者的我不得不采取行动了。。。还好有我的AI同事！

**2/ 探索**

于是开始提问：

> ```
> This template is used to generate alpha templates that have economical meanings, you will run a deep analysis on this template, you shall 1) find out the ts op combo that clearly not making sense 2) split the template, and operator dict, if necessary, to make this template into meaningful buckets. You will run the analysis by me first, DO NOT EDIT THE CODE without asking me
> ```
> 中文prompt：
> ```
> 这个模板用于生成具有经济含义的阿尔法模板。你需要对其进行深入分析，任务如下：
> 1）找出那些明显没有意义的 ts 操作组合；
> 2）如有必要，将模板和算子字典拆分成有意义的“桶”。
> 整个分析过程须先与我确认，**未经我允许不得修改代码**。
> ```

以下是AI的解答（只摘取了重点部分）。
 *_心得：和AI对话不仅是为了获取结果，还是为了学习和借鉴它的分析思路。后者的影响可能更为深远。_*

**第一步：** 了解各个operator的信息，只有这一步用到了mcp。

**第二步：** 找出明显不符合逻辑的operator组合，
比如：
- Timing-of-timing: `ts_arg_min(ts_arg_max(...))` - AI认为这个没有意义
- Correlation operations: `ts_corr(...)` - 找出来一个bug：这个操作符需要两个series，不符合原模板
- Redundant normalizations: `ts_rank(ts_zscore(...))` - 这两个都是做normalize工作，重复了

**第三步：**

对operator进行分类：

- 1. 差分/变动类算子（diff_op)：`ts_delta`, `ts_av_diff`, `ts_min_diff`, `ts_min_max_diff`
- 2. 归一化/排序类算子 (norm_op)：`ts_rank`, `ts_zscore`, `ts_median`, `ts_std_dev`, `ts_skewness`
- 3. 窗口聚合类算子 (agg_op)：`ts_sum`, `ts_decay_linear`, `ts_decay_exp_window`
- 4. 位置/索引类算子 (pos_op)：`ts_arg_min`, `ts_arg_max`, `ts_delay`
- 5. 特殊算子 (special_op)：`ts_count_nans`, `ts_entropy`, `inst_tvr`, `ts_min_max_cps`

**第四步：**

研究operator的组合能否产生有经济学意义的模板。AI给出了以下推荐：

**模板1 动量加速（差分→差分）** 
"{diff_op1}({diff_op2}({x}, {d1}), {d2})"
经济含义：捕捉趋势的加速或减速  
算子：ts_delta、ts_av_diff、ts_min_diff、ts_min_max_diff

**模板2 标准化动量（标准化→差分）** 
"{norm_op}({diff_op}({x}, {d1}), {d2})"
经济含义：对动量信号做排序或标准化  
标准化算子：ts_rank、ts_zscore、ts_median、ts_std_dev、ts_skewness  
差分算子：ts_delta、ts_av_diff、ts_min_diff、ts_min_max_diff

**模板3 聚合信号（聚合→标准化）** 
"{agg_op}({norm_op}({x}, {d1}), {d2})"
经济含义：对标准化信号做平滑或时间衰减  
聚合算子：ts_sum、ts_decay_linear、ts_decay_exp_window  
标准化算子：ts_rank、ts_zscore、ts_median、ts_std_dev、ts_skewness、ts_entropy

**模板4 位置信号（位置→聚合/标准化）** 
"{pos_op}({inner_op}({x}, {d1}), {d2})"
经济含义：找出极端值相对于聚合信号出现的位置  
位置算子：ts_arg_min、ts_arg_max、ts_delay  
内部算子：ts_rank、ts_sum、ts_zscore、ts_std_dev
