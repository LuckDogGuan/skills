## **1.自我介绍**

先自我介绍一下，我是2024年10月13日参与上海线下培训，当天下午提交了平台上的第一个alpha，10月18日（花了五天）满足10000分收到顾问邀请，10月25日（花了7天）获得顾问权限，当天交了第一个顾问alpha。

## **2.常见踩坑大全**

### 2.1. 更新时间问题

WorldQuant平台上默认有超过3种时间，算上北京时间就是四种时间。
①提交alpha的时间节点是美东时间（UTC-5），也就是北京时间13:00为一天的分界线。
②Atom比赛用美国中部时间（UTC-6），也就是北京时间14:00为一天的分界线。
③alpha的Properties里面默认是UTC+13时间，比北京时间快了5小时。

### 2.2. 挑选数据集

目前应该大家都用的是machine_lib的一套代码，有一阶段、二阶段和三阶段。
那么每一个数据集都质量参差不齐，所以良好的数据集应该是三阶段数量≥二阶段数量≥一阶段数量。
如果跑了10000个一阶段发现只有100个二阶段，那么赶紧换数据集吧!

### 2.3. pyramids问题

pyramids是一个很重要的指标，是成为expert、master、grandmaster的必要条件。
但是新手往往都会集中在USA，这里我建议大家不要集中在USA，或者是TOP3000，因为所有人都在跑，prod correlation会无法通过的，可以试试ASI、EUR之类的。

### 2.4. 为什么我达到了expert或者master的要求但是却没有升级

年底12.31才会分流。所以即使你达到了expert或者master的要求，也不会立刻升级，只有等到年底才会升级。
而且每个level的人数是有限的（expert、master、grand master分别为30%，8%，2%），如果达到了上限就会按照排名来划分。
*每个Alpha的平均不同操作符数量（越低越好）
*每个Alpha的平均不同字段数量（越低越好）
*总不同操作符数量（越高越好）
*总不同字段数量（越高越好）
*社区领导者
    *论坛：在季度末发布的帖子或评论的点赞数量。帖子或评论的最小长度必须为100个字符。
    *推荐：季度内成功推荐的总数。当推荐的顾问根据推荐费计划的指南，成为有资格积累推荐费的人时，推荐被视为“成功”。
*最长连续回测天数：顾问在该季度结束或持续到下一个季度的最长连续回测天数。不会每季度重置为0。

### 2.5. 什么样的因子适合提交？

详情还是可以看2.4.的排名，WorldQuant给出了他认为好的。他希望你的alpha是独特的，用到的fields和operator越多越好，但是单个alpha用到的fields和operator越少越好，因为可能会过拟合。
而且如果想成为expert、master、grand master就需要OS的sharpe达到0.5、1.0、2.0，因此你提交的alpha的IS夏普尽量要在2.0以上，而且不是过拟合的，这样OS才有可能满足要求。

## **3.如何提高base pay**

目前我们的value factor都是0.5，所以质量不高的因子的base pay只有1.5美元。
想提高base pay就需要提交OS表现好的alpha来提高value factor。
短期来看的话可以参与theme，主题类型的因子会有2.5倍的bonus，我之前最高一天拿到了4.48美元，交了三个主题因子。
目前（2024年11月18日）是CROWDING主题，大家可以在平台的邮箱里看到，也可以尝试一下。
另外一个就是pyramids的主题，在genius里能看到一张很大的乘数表格，越高的给的应该是越多的。

### 评论 #5 (作者: XX42289, 时间: 1 year ago)

### **2.7 一阶段、二阶段和三阶段**

回答FP33939，一阶段、二阶段和三阶段是线下培训提出的概念，可以在群里要一下今年最后一次培训的报名表，里面会提到的。
这是一种类似于遗传算法的思想，但是是青春版遗传算法。类似于优中选优。

你先用原始的数据字段跑回测，会得到一阶段的数据里Sharpe比较好的，比如选出一阶段夏普大于等于1.0的

然后你用这些一阶段的优秀的字段，套上其他的ops，比如group_ops，再重新回测，会得到一批二阶段的alpha，然后在里面挑选好的，比如夏普大于等于1.3的

然后用这些二阶段的优秀字段，套上trade_when，ifelse等等ops进行进一步的择时，重新回测，会得到一批三阶段的alpha。这里面由于是优中选优，所以大部分都是可以交的alpha，只是有可能不能过prod corr。

以此类推，四阶段五阶段都可以一直下去，但是要谨防过拟合！

### 评论 #6 (作者: XX42289, 时间: 1 year ago)

### **2.8 负号的坑**

有很多同学发现有的alpha回测完以后夏普是负的，而且很高，比如-2.0，那么只要在表达式加上一个负号(-)，就可以让他编程夏普2.0。这是一个常见的改进方法。

但是如果需要程序去自动加负号的话，一般而言是直接在表达式的前面加个'-'负号就行了。

例如 :

```
group_zscore({alpha1}, densify({alpha2}))
```

可以变成：

```
-group_zscore({alpha1}, densify({alpha2}))
```

但是有一种情况不能这样，就是你在使用模板的时候

例如

```
grouped = group_zscore({alpha1}, densify({alpha2}));ts_mean(grouped, 120)
```

这种情况下如果你直接在前面加个负号，就是不对的。正确应该在  **ts_mean** 前面加负号，而不是  **grouped**

```
grouped = group_zscore({alpha1}, densify({alpha2}));-ts_mean(grouped, 120)
```

**但是！！保险起见，** 请加入一个括号后再添加负号，如下：

```
grouped = group_zscore({alpha1}, densify({alpha2}));-(ts_mean(grouped, 120)
```

为什么呢？上式加不加括号都一样，但是如下的式子就一定要加括号，可以自己理解一下，灵感来源于HJ33503

```
grouped1 = group_zscore({alpha1}, densify({alpha2}));grouped2 = group_zscore({alpha3}, densify({alpha4}));cond = grouped1 > grouped2-(cond ? grouped1 : grouped2)
```

```
grouped1 = group_zscore({alpha1}, densify({alpha2}));grouped2 = group_zscore({alpha3}, densify({alpha4}));-(grouped1 - grouped2)
```

### 评论 #12 (作者: BZ29383, 时间: 6 months ago)

既然楼主提到了前期的奖励，我就跟个风吧！我之前在直播听课的时候，听到老师将的，我做了个总结！

第一、短期赚钱可取，比如压主题可以额外获得双倍奖励，不同分类的主题的本身的倍率也不一样！

第二、对于每日奖励，实际上是看平均分的，如果你提交1个10分的alpha，那么你的奖励分是10分。如果你提交的2个alpha，1个10分，另1个5分，那么你的奖励分可能就变7.5了。所以如果你是为了短期奖励，那就尽量选最好的alpha提交。

第三、关于VF，其实平台还是挺严格的。一个是多样性，就是要多区域多分类数据集中去点塔；一个是可扩展性，所以一般单个alpha提交尽量要求纯净，也就是ATOM Alpha，这样后面组alpha的时候更好。有时候不要为了点塔而选择混PV 混Other等等。虽然你可以一次性点2个塔，但是后面不好扩展。还有就是对于PC的关注，很多人盲目追求低PC却忽视了alpha的质量，首先是这个alpha质量高然后才看PC，为啥PC低你要注意看了，可能就是因为这是别人不要的！

第四、对于各个参数要有一定的了解，可以看看近10年的数据走势，比较典型的就是“厂”字型，一般2Y sharpe都不过关，近几年的表现不太好。还有就是换手率turnover，在IND地区手续费较高，你高换手率就意味着高交易，每次手续费也是个不小的成本，所以也很重要。当然还有margin，大部分地区提交的时候尽量保证万10的数据，个别区域更是需要万15以上。

以上仅是我3个月以来，总结的一些经验，希望可以和大家一起进步！！
