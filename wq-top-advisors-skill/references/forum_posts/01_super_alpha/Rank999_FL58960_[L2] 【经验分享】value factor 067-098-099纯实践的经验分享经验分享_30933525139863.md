### 前言

各位好，我是去年10月份线下进行培训的那一批的学员，先后经历了3次的Value Factor的更新，分别是0.67——>0.98——>0.99。这个变化完全是出乎自己的意料，诚惶诚恐。因为自己之前没有金融相关的经验，现在也是当社畜没有很多时间来研究这方面。所以本次的经验分享完全基于自己在这几个月的实践过程中一些心得体会，如果有不对的地方大家可以评论区指正，希望我的完全感觉性的经验能够多多少少帮助到大家。

### 1、最核心的分享——只做Single Data Set Alpha（ATOM）

为什么我最开始的Value Factor只提升了一点捏？就是因为第一个月我年轻不懂事，一阶不行就二阶，二阶不行就三阶，三阶还不行就再次嵌套一二三阶，这样子做首先操作符的数量会直线上升，我记得上个季度我的平均ops最开始好像是9点几，其次就是引入了很多别的数据，非常有可能过拟合。最后就是alpha的质量可能非常不稳定，差的可能os就直接爆炸了。

然后我参加的第一次研究小组会议就听到了HuaHong大佬（@HZ69256）的分享，当时他就介绍说他只做Single Data Set Alpha，然后提到了据他观察Single Data Set Alpha的表现是要比混信号的Alpha稳定的。从那次以后，从去年12月开始到现在，我基本上就是只做Single Data Set Alpha了，如果某一天找不到Single Data Set Alpha了，我宁愿断粮，那一天就不交了。我个人感觉后面两次的Value Factor更新能有很大的提升最核心的原因可能就是我只交Single Data Set Alpha，这也算是我个人最大的一个特点了。

### 2、个人做Single Data Set Alpha的步骤

因为本人对金融方面的知识一窍不通，然后又是社畜的关系平时能够仔细研究平台上各类知识经验的时间比较少，所以我每次做alpha的步骤完全是基于感觉来的。不过这一套的步骤我倒是化为体系了，在这里分享给大家供大家参考：

**第一步：首先确定目标数据集** ：这一步就是大家选一个数据集，这个可以基于自己想要刷py，也可以基于自己以往做alpha的经验选择相对好出alpha的数据集类型。当然选数据集的要看一下这个数据集有没有人出过alpha，要是 **Alpha** 那一项是0那就不要考虑了，我自己尝试过几次这种的数据集，基本上就是没啥出alpha的机会。

**第二步：跑一阶** ：选好数据集后就可以开始一阶了。一阶的模板我是这样选择的，首先最开始培训的那套代码还是可以选择的，不过可能现在应该要放在那些Value Score稍微高一点，Alpha数量较少的数据集。如果实在是感觉最开始培训那套一阶方法Prod太高了，那就换自己的模板。我的模板是社区的模板分享里推荐的，不过我会拿过来魔改。魔改的方法是只保留最核心的2-3个操作符，然后保留一个数据字段field的符号（op2（op1（field））大概这样子），后面一阶就是把选定的数据集的数据字段都填充进去遍历回测就行。我个人感觉一阶的模板越简单越好，复杂的模板我也尝试过，但是感觉有点难出货（可能运气不好没有选到合适的）。后面我就自己魔改了，反正魔改的越简单越好，我自己感觉越简单的出货概率越大。不过这个策略仅供参考，不要迷信，还是那种有经济学意义的模板更好。

**第三步：选取表现较好的一阶alpha&Neutralization** ：一阶全部都回测完了就开始筛选了，我的筛选的话还是还是先以fitness为准，首先各项指标不异常的情况下（就是其他的指标不出现什么极值情况，比如weight too concentrate，2 year sharpe=0这种）按照fitness从高到底筛选(个人的阈值是fitness>0.9,sharpe>1.4，不行就换数据集再跑一阶），然后选择表现最好的那个一阶alpha，开始测试不同的Neutralization。

**第四步：基于选取一阶alpha&Neutralization开始跑二阶** ：在我把一阶的相关指标都确定好一个大概后，就开始二阶，把一阶筛选好的alpha取出来然后进行二阶，二阶我也是用group操作符，不过我稍微做了一下修改，主要是把group的字段换成了single_groups = ["market","sector", "industry", "subindustry", "country"]，不然没有办法做Single Data Set Alpha。然后查看回测的结果。回测的结果如果还是不行或者差一点的话，我就再做一次group或者signed_power(x,2)、signed_power(x,0.5)（研究小组的大佬们的办法）。如果还是不行的话，我就放弃这个数据集了，因为在再套可能有过拟合的风险，而且平均ops也会上升。

### 3、SuperAlpha的分享：像Regular一样用ts操作符

SuperAlpha这块我最近也探索了一下，主要是我发现用等权的办法相关性太高了，没有办法交捏。所以我就捣鼓了一下想要把combo可以使用的操作符也用上，想要突破这个相关性。以下是代码（我弄的比较晚，可能大家都已经捣鼓过了哈哈哈）

```
super_states = ["stats.returns", "stats.pnl", "stats.turnover", "stats.drawdown", "stats.hold_pnl",                "stats.hold_shares","stats.hold_value", "stats.long_count", "stats.long_value",                "stats.short_count", "stats.short_value", "stats.trade_pnl", "stats.trade_shares" ,"stats.trade_value"]​#ts操作符，换成自己的能用的combo_ops = ["combo_a","signed_power","ts_rank","ts_arg_min", "ts_arg_max",          "ts_max_diff", "ts_returns","ts_zscore", "ts_delta",  "ts_sum", "ts_delay",          "ts_ir", "ts_std_dev", "ts_mean", "ts_scale", "ts_kurtosis",            "ts_quantile","last_diff_value","ts_av_diff","ts_product"]​days = [5, 22, 66, 120, 240]​#super的combo表达式生成def ts_super():    combinations = product(super_states, combo_ops, days)    combo_a_modes = ['algo1', 'algo2', 'algo3']        expressions = []    # 先处理 combo_a    for mode in combo_a_modes:        expr = f"combo_a(alpha, nlength=250, mode='{mode}')"        expressions.append(expr)        # 再处理其他操作符    for super_combo, ts_op, window in combinations:        if ts_op != "combo_a":  # 跳过 combo_a            expr = f"stats=generate_stats(alpha);{ts_op}({super_combo}, {window})"            expressions.append(expr)        return expressions
```

这样就可以得到combo表达式的列表，然后将回测的代码变成superalpha的格式（参考acelib.py文件），然后将表达式进行回测就行（就和regular一样的方式，只不过是combo表达式），只不过superalpha只能同时回测3个。

您好，我是这样的，因为每次回测的alpha都是一个数据集，那么我在这个数据集里面的先挑表现比较好的，比如Sharpe>1.4,fitness>0.9，这里面可能会有很多的alpha，但是因为他们是同一个数据集的，所以大概率同样的neutralization会是相同的效果，所以我选表现效果最好的那一个alpha，在网页上手动测试所有的neutralization，选择最好的那一个，然后二阶的回测使用的neutralization就是选择的最好的那一个，这样就没有什么数量级的问题了。当然这个做法不一定有效果，可能同一个数据集的数据在不同的表达式的情况下，我这种办法不一定选的出适合的neutralization，但是据我的实践经验来说，一般都是可以使用的，你可以根据你的情况来调整。同时如果说感觉一阶好的alpha特别多，你全部筛选出来，你可以按照操作符和数据字段进行一下分类，每次只跑一个数据字段或者一种操作符的二阶，这样可能会减少数量级
