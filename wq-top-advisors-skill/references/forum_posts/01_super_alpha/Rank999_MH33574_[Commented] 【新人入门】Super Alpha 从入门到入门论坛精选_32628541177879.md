## SuperAlpha 理论与实操指南：从单因子走向系统组合

## 一、什么是 SuperAlpha？

在 BRAIN 平台上，每个 Regular Alpha 是一个策略信号，而 SuperAlpha 则是通过系统性逻辑表达，把多个 Alpha 策略按一定权重组合，构成一个更稳定、更强大的复合策略。

构建 SuperAlpha 有助于：

- **提高整体稳定性** ：降低单一 Alpha 波动风险；
- **改善评分表现** ：提升 VF、Combined Score 等关键指标；
- **增强盈利能力** ：获得额外 $1–$60 的基础奖励；
- **扩大参与边界** ：通过组合构建方式，进入更复杂市场结构；
- **升级顾问等级** ：构建 SuperAlpha 是通往高级顾问权限的核心门槛。

SuperAlpha 是策略研究从“因子开发”走向“系统构建”的关键一步。

## 二、SuperAlpha 的构建结构

SuperAlpha 包含两个核心表达式：

- **Selection Expression** ：选择哪些 Alphas 进入组合；
- **Combo Expression** ：决定这些 Alphas 如何每天按比例加权组合。

逻辑流程如下：

1. 使用 Selection Expression 为每个 Alpha 计算一个分数；
2. 系统按分数高低排序，选出前 N 个 Alphas（由 Selection Limit 决定）；
3. 使用 Combo Expression 为这些 Alphas 每天分配组合权重；
4. 输出结果构成最终 SuperAlpha。

## 三、Selection Expression 原理详解

### 核心机制：排序与筛选结合

Selection Expression 的本质不只是筛选器，而是一个 **筛选+打分函数** 。系统对每个符合条件的 Alpha 计算表达式数值，称为  **selection weight** ，然后按照该值从高到低排序，选出前 N 个 Alphas，N 由  **回测设置中的  `Selection Limit`**  控制。

这一机制意味着：

- 表达式结果可以是布尔值（1/0），也可以是连续评分；
- 排名靠前的 Alpha 才会被选入组合；
- `Selection Limit`  是一个“名额阈值”，控制最终入选 Alpha 的数量。

### 示例表达式对比：

类型
表达式
说明

布尔型

 `turnover < 0.2` 
所有符合条件的 Alpha 得分为 1，其余为 0，系统在所有 1 中取前 N 个

连续评分

 `1 - self_correlation` 
Alpha 越不相关，得分越高，更容易入选

组合表达

 `(fitness > 2) * (1 - prod_correlation)` 

 `fitness`  是门槛， `prod_correlation`  决定排序优先级

多因子加权

 `0.5 * (1 - turnover) + 0.5 * (1 - self_correlation)` 
自定义加权评分，综合多个维度

### Selection Handling Setting里的部分参数：

- `Positive` （默认）：只保留 > 0 的得分；
- `Non-Zero` ：保留所有非零值；
- `Non-NaN` ：只剔除 NaN，适用于复杂逻辑表达式。

## 四、Combo Expression：构建权重逻辑

Combo Expression 用于为选中的 Alpha 分配每日权重。

系统会将选中的 Alphas 表达式输出按  `(A x D x I)`  维度组织成矩阵，Combo Expression 的结果是一个  `(D x A)`  的权重矩阵。最终组合输出是加权后的 Alpha 信号总和。

### 常见表达式示例：

用法
表达式
说明

等权组合

 `1` 
每个 Alpha 权重相等

IR 加权
 `ts_ir(generate_stats(alpha).returns, 250)` 
最近一年表现好的 Alpha 权重更高

稳健性加权

 `1 - reduce_max(self_corr(generate_stats(alpha).returns, 500))` 
高相关 Alpha 权重降低

低换手惩罚

 `ts_ir(stats.returns, 250) * (1 - stats.turnover)` 
结合表现与换手率优化组合结构

## 六、Global SuperAlpha 构建要点

构建 Global SuperAlpha 时需特别注意区域间中性化与数据质量问题：

- `neutralization = COUNTRY`  是推荐的；
- 推荐在  `TOP3000`  上先调试，再迁移至  `MINVOL1M`  回测提交；
- 精选 Neutralization 设置为 COUNTRY 且 OS 数据完整的 Alpha：

```
neutralization == "COUNTRY" && os_start_date <= "2022-01-01"

```

- `combo_a`  运算符适用于组合差异化高的 Alpha；
- 开启  `max_trade` ，确保策略在流动性约束下具备实际可投资性。

- `not(own)` ：排除自有，只用外部池（需有配额）；

## 八、实用设计建议

1. 使用标签系统（tags）管理 Alpha 分类，提升表达式可读性与管理效率；
2. 为每个 Combo 测试等权组合作为基线，验证组合表达式是否提升有效性；
3. 多使用  `generate_stats()`  和  `self_corr()`  构建与表现、稳定性相关的动态加权；
4. 通过 Run Selection 功能快速验证表达式逻辑是否有效；
5. 在 Selection 中结合布尔门槛与连续排序因子，提高选中 Alphas 的质量；
6. 针对不同市场区域、风格因子、数据深度构建多个 SuperAlpha，提升组合多样性。

## 九、结语

SuperAlpha 是平台对策略构建能力的深度检验。它不仅关乎“因子表现”，更反映了研究者在组合稳定性、风险控制、多因子整合方面的综合能力。

真正优秀的 SuperAlpha 不仅能在 IS 和 OS 表现良好，也具备跨周期、跨行业、跨区域的适应性。建议所有中高级顾问系统性地投入 SuperAlpha 构建，以策略组合的方式提升整体研究价值，解锁更多平台权限与收益可能。

构建SA的时候尤其要注意信号的衰减，如果sharpe在近两年有较大的衰减，则可能是信号失效的显著表现。
