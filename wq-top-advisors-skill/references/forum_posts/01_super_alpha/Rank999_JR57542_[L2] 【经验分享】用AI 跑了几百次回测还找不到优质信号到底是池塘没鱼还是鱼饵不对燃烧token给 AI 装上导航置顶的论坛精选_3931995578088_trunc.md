## 一、我的工具链背景

我是计算机专业出身，目前用 Claude Code + Skills 的方式让 AI 全程自动化地搜索和迭代 alpha，核心思路是：通过海量 token 消耗换取高度自动化的探索过程，减少人工干预，让 AI 自主跑完"灵感生成→回测验证→迭代修正"的完整闭环。

## 二、现状和痛点——为什么要优化？

不管是通过 cnhkmcp 找灵感，还是用 AI 打工人体系里的一套现成方法，用下来都有三个问题层层递进：

**第一层：模板错配。**  现成模板未必适配你正在探索的数据集特征，同类模板被大量使用导致同质化。AI 反复回测没有信号的表达式，资源浪费严重。

**第二层：AI 的"贪心病"。**  不加引导的 AI 天生倾向于走贪心路线——局部最优步步为营，结果越走越窄，几轮回测失败之后就"失去信心"，直接下判断"本数据集失败了"，然后放弃。这是典型的一叶障目。

**第三层：根本矛盾。**  以上两层问题的背后，有一个更本质的困境： **AI 无法区分"是表达式写得不对"还是"数据字段本身质量不够"** 。这两种情况在回测结果上看起来一模一样——Sharpe 都很低，AI 都不知道该继续深挖还是换方向。

想象一下这场景：AI 自主挖掘的过程，就像开着没有导航地图的宇宙飞船飞向浩瀚太空寻找宝藏星球——既然都是乱开，为何不做一下设计，让自己的航线更"高效"？

## 三、原版 GEM 工作流——好在哪里，卡在哪里

原版 GEM 工作流的流程如下：

```
拉取数据集所有字段  → 第一步：做归类分类  → 第二步：生成基于字段描述和分类的 idea 灵感文件  → 第三步：引导 AI 基于 idea 文件批量写出回测表达式
```

**优势** ：鼓励 AI 创建纯粹的、基于裸数据字段加减乘除的表达式，通过回测"测试"信号是否存在；idea 文件也给后续探索赋予了经济学故事的合理性。

**痛点** ：用下来很容易出现的情况是——token 消耗飞起，但 AI 死活找不到，尤其遇到数据集比较冷门的时候，AI 很快就会放弃，直接宣布方向失败。根本原因是： **GEM 工作流没有在 AI 的灵感和真实数据字段的性质之间搭起桥梁** ，单纯的 GEM 一把梭，不过是 AI 幻觉，是空中楼阁。

## 四、我的三板斧魔改

### 第一斧：论文预搜索机制

在写 idea 之前，先要求 AI 根据拉取下来的数据字段描述的关键词，上网搜索 20 篇相关学术论文，优先找近年的（近年论文更适应市场现状，太老的 alpha 可能已经被市场充分消化了）。

这一步的作用是：给整个流程搭上更厚重的经济学基础，让后续生成的表达式不再是 AI 凭空幻想，而是有学术依据支撑的。

### 第二斧：校准策略（螺旋迭代）

一次 GEM 生成表达式后，不要直接"一把梭"全部回测。我的做法是：

```
批量生成 ~200 个表达式  → 批量回测  → AI 分析这 200 条结果，找出"哪些方向有苗头、哪些方向全军覆没"  → 反过来修改 idea 文件，调整灵感方向  → 再生成下一批 ~200 个表达式  → 螺旋迭代 1-2 轮
```

这样做的关键价值在于：用真实回测结果给 AI 一面镜子，让 AI 通过数据自我纠偏，而不是靠一开始的灵感一路盲走到底。

### 第三斧：信号灯系统（Direction Radar）

前两板斧解决了"灵感从哪来"和"怎么迭代"的问题，但还有一个根本问题没解决—— **AI 跑了几轮回测之后，到底应该继续深挖这个方向，还是及时止损换方向？**

这个判断之所以难，是因为存在一个核心的混淆变量：

> 回测 Sharpe 不好的时候，你不知道到底是"数据字段本身没信号"，还是"你的表达式没把信号挖出来"。
> 就好比钓了半天没钓到鱼，不知道是这个池塘根本没鱼，还是你的鱼饵不对。

为了解决这个问题，我设计了一套"信号灯系统"。核心思路是： **不只看单次回测结果好不好，而是用统计学方法分析一整批回测结果的分布特征，给出方向性判断。**

#### 怎么判断亮哪盏灯？

系统会从四个维度对每批回测打分，加权算出一个综合评分：

1. **信号显著性（权重最高）** ：用 t 检验看这批 Sharpe 均值是不是真的高于噪声线——通俗地说就是：这批表达式的表现到底是真本事还是瞎猫碰死耗子？如果样本太少，就换用 Bootstrap 重采样来估计，避免在数据不足时瞎下结论。
2. **天花板高度** ：看这批里最好的那个表达式能到多好。哪怕均值不高，但如果最好的那个个体 Sharpe 很亮眼，说明这个方向的结构是有潜力的，只是大多数表达式还没写对——这种情况不应该轻易放弃。
3. **提交通过率** ：这批里有多少能通过平台提交检查，用置信区间处理小样本情况（比如只回测了 10 个通过了 3 个，不能简单说通过率 30%，需要估计置信区间，统计学上叫 Wilson Score）。
4. **稳定性** ：看 Sharpe 值的离散程度。结果忽高忽低，信心要打折扣；结果集中稳定，说明方向靠谱。

#### 解决"钓鱼问题"的关键——算子多样性分数

这是整个系统里我觉得最有意思的设计。

我把 BRAIN 平台的算子分成了 6 大族： **时间序列类** （ts_mean、ts_decay_linear 等）、 **截面分组类** （group_zscore 等）、 **数学变换类** （log、abs 等）、 **条件布尔类** （if_else 等）、 **全局排名类** （rank、percentile 等）、 **滞后移位类** （ts_delay 等）。

然后用这个指标来区分"没鱼"和"鱼饵不对"：

- **用了 4 种以上算子族，结果还是不行**  → 大概率真的是"池塘没鱼"（数据字段本身质量不够），系统会更快地判 DEAD，避免继续浪费回测配额。
- **只用了 1-2 种算子族，结果不行**  → 可能只是"鱼饵太单一"，系统会要求你先拓宽探索范围再下结论，不会过早放弃。

这一设计本质上是在控制一个混淆变量——低 Sharpe 的锅，得先搞清楚是数据的锅还是算法设计的锅，才能对症下药。

#### 5 条防误杀护栏

判 DEAD 是最危险的决策——一旦错判，就会错过一个有价值的方向。所以我设了 5 条护栏，核心原则是 **宁可多探索，不过早放弃** ：

1. **小样本保护** ：回测不到 5 个？永远只给黄灯。不到 10 个？不允许判红灯或死灯。数据不够就不做强判断。
2. **天花板保护** ：如果最好的那个 Sharpe 达到了一定高度，就算均值很低也不判死灯——有一个成功个体，就说明方向有潜力，只是大多数表达式还没找到正确的打开方式。
3. **双峰保护** ：如果一批回测的 Sharpe 分布是双峰的（一群高分一群低分），说明批次里有个高分子群体可以提取，不应该因为整体均值拖后腿就放弃整个方向。
4. **三重证据门槛** ：判 DEAD 必须同时满足三个条件——样本量够大 AND 均值够低 AND 最大值够低，三者缺一不可。任何单一条件都不足以宣判。
5. **趋势保护** ：如果最近几轮迭代的均值在上升，说明你正在进步中，不会因为当前绝对值低就降级判定。

#### 额外能力

除了单批分析，系统还有两个附加能力：

- 用  **Mann-Whitney U 检验** 对比两轮回测，统计上判断这次结构改动是不是真的比上次有提升（而不是靠感觉说"好像好了一点"）。
- 用  **Spearman 秩相关** 追踪整个迭代历史，判断方向是在收敛、震荡还是在退步，帮助决定是继续当前路线还是需要大幅转向。

#### 验证

我在 24 个历史迭代批次上做了回顾性验证：准确率 95%，最关键的是 **假阴性为 0** ——没有一次把有信号的方向误判成了 DEAD。

## 五、魔改后的完整工作流

三板斧整合后，完整的流程变成了：

```
拉取数据集所有字段  → 做归类分类  → 【第一斧】搜索 20 篇相关论文，构建经济学基础  → 生成基于字段和论文的 idea 灵感文件  → 批量生成 ~200 个回测表达式  → 批量回测  → 【第二斧】分析结果，修正 idea 文件，螺旋迭代 1-2 轮  → 后续自由探索迭代过程中：      【第三斧】信号灯系统实时导航        🟢 绿灯 → 加码深挖        🟡 黄灯 → 谨慎继续，尝试结构变体        🔴 红灯 → 做一次结构性改动        ⚫ 死灯 → 及时止损，换方向
```

统计学原理支撑                             

  信号灯系统背后基于以下数学公式：             

  ① 单样本 t 检验（判断均值是否显著高于噪声线）                                                       

  t = (样本均值 - 1.0) / (样本标准差 / sqrt(样本数量))                                                

  以 Sharpe = 1.0 为零假设（方向无信号），计算单侧 p 值。p < 0.05                                     
  时判定方向有统计显著信号。当样本量不足 8 个时，改用 Bootstrap BCa（偏差校正加速法，9999
  次重采样）代替 t 检验，避免小样本下的误判。                                                         

  ② 双峰系数 BC（识别批次内是否存在高分子群体）                                                       

  BC = (偏度^2 + 1) / (峰度 + 3*(n-1)^2 / ((n-2)*(n-3)))                                              

  BC > 0.556 时提示分布可能为双峰——说明批次里存在一个高分子群体，不应因整体均值低就放弃整个方向。     

  ③ DSI 方向强度指数（综合评分）                                                                      

  DSI = 0.30 * S_ttest + 0.25 * S_ceiling + 0.25 * S_passrate + 0.20 * S_consist                      

  各分量定义：                                                                                        
  - S_ttest    = clip(1 - p值, 0, 1)，即 t 检验的显著性得分
  - S_ceiling  = min(1, 批次最高Sharpe / 2.0)，即天花板高度得分                                       
  - S_passrate = 批次内通过提交检查的比例                      
  - S_consist  = 1 / (1 + 变异系数)，变异系数 = 标准差 / |均值|，越稳定得分越高                       

  ④ Mann-Whitney U 检验效应量（对比两轮迭代谁更好）                                                   

  效应量 r = 2 * U1 / (n1 * n2) - 1                                                                   

  r > 0 表示新一轮更好，r < 0 表示旧一轮更好。|r| > 0.3 为中等效应，|r| > 0.5                         
  为大效应。非参数检验，不要求正态分布，对小样本友好。

  ⑤ Spearman 秩相关（追踪迭代是否在收敛）                                                             

  rho = 1 - 6 * sum(秩差^2) / (n * (n^2 - 1))                                                         

  对各轮迭代的均值 Sharpe 序列计算秩相关：rho > 0.5 且 p < 0.05 判定为收敛；rho < -0.3                
  判定为发散，提示需要大幅换方向。没必要再浪费token和回测时间砸进去了。

## 六、效果和代价

目前这套系统每天消耗约 500 次回测，能产出一个 regular alpha 并且完美符合平台标准。

代价是时间：往往需要几个小时——回测本身的等待时间 + 拉取数据、检测 PC/SC 的等待时间叠加在一起，很难快。这也是为什么开头要做 token 消耗警告——高自动化的代价，是高消耗。

但和手动迭代相比，AI 全程自动化的探索覆盖面要宽得多，而且有了信号灯系统之后，不会再出现 AI 死磕一个方向死活不换的情况了。
