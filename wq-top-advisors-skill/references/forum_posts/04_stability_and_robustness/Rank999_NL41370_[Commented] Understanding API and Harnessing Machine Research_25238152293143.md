## 帖子正文

API (Application Programming Interface) is like a mutual language that computers use to talk to each other. For the BRAIN platform, the API lets you fetch data and send necessary information to us, bypassing the need for the User Interface (UI).

Sounds great, right? But, what's even better is leveraging the power of automation in your research (often referred to as "machine research")!

**Machine Research vs Human Research**

Normally, you'd write your alpha expression in the Simulate tab, click the Simulate button, and wait for the PnL chart and summary. But with the BRAIN API, you can consistently send alphas for simulation and later analyze the results. This method saves valuable time and is more efficient.

**When to Use Machine Power?**  🤔

While it's suggested to build alphas grounded in sound economics, repetition can be automated. You can leverage machine power in your alpha research when:

- You want to test different implementations of an idea. 💡
- You aim to optimize parameters like decay, neutralization, or lookback periods. Remember, a good alpha isn't sensitive to small parameter changes. ⚖ small changes in parameters or arbitrary values should not be the main determinant of an alpha performance. So it’s always a good practice to choose sensible parameters values like 5, 20, 60, 252 for lookback days or 0.01 for truncation.
- Running statistical analyses on a batch of alphas. Our platform provides endpoints for important alpha stats. ⚙️

Testing your alphas' robustness  [**🛡**  **️**](https://emojipedia.org/shield/)  before submission. You can design tests to automatically check this!
