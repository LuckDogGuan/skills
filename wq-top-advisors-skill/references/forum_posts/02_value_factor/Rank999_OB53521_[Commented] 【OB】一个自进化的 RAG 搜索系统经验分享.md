## 1. 简介

这是一个给 AI Agent 用的 RAG 搜索系统，主要是为了解决我自己在 alpha 挖掘过程中的一个痛点。

GitHub地址： [https://github.com/BeastOrange/wq-forum](https://github.com/BeastOrange/wq-forum)

平时做研究或者调 alpha 的时候，经常会遇到需要查资料、翻论坛、找历史讨论的情况。直接网页搜索有时候搜不到，搜到了也经常不够准；更麻烦的是，同一个问题过一段时间又会再遇到，还是要重新搜一遍、重新判断一遍。

所以我想做一个更适合 Agent 使用的本地搜索系统：它不只是把论坛内容做成索引，然后给出几个搜索结果，而是可以在使用过程中把有价值的结论沉淀下来。下次再问类似问题时，系统会优先使用已经整理好的知识，再回到原始帖子里查证。

这个想法参考了一篇公众号文章里提到的“自组织 / 自进化”知识系统

## 2. 这个系统主要做什么

目前这个项目做的是一个本地离线的论坛 RAG 系统。它可以把论坛导出的 JSON 数据解析出来，存进 SQLite，然后提供搜索、查看原帖、精确查找、相关帖子推荐等功能。

这部分已经比较实用了。比如我想查某个 alpha 相关概念、某个 operator、某段报错或者某个历史讨论，不需要再手动去网页里翻，而是可以直接让 Agent 调用工具搜索。

比较关键的是，这个系统不是给人手动点来点去用的，而是给 AI Agent 用的。我把搜索、读取原帖、沉淀知识、检查知识页、建立知识关系这些能力都做成了 MCP 工具。这样 Gemini CLI、Codex 或其他支持 MCP 的 Agent 都可以直接调用。

我简单使用了一下：让AI设计一个较为全面的一二三阶代码，大概10秒钟之后，我得到了高度相关的结果。全新的对话，没有对任何名词的解释，都是AI自己搜索整理结果。

## 3. 为什么说它是“自进化”

这里的“自进化”没有接入LLM API， **因为我不想花钱。**

我的设计是：Agent 负责阅读、判断和总结；系统负责提供可靠的工具和存储层。

比如 Agent 搜索到几篇帖子后，如果发现里面有一个可以复用的结论，就可以把它整理成一篇知识页。写入知识页时必须带上来源 topic id，也就是这个结论是从哪些原帖来的。系统会检查来源是否存在、置信度是否足够、有没有冲突关系。

它的进化过程大概是：

1. 第一次遇到问题，Agent 从原始论坛资料里搜索和判断。
2. 如果结论有复用价值，Agent 把它保存成有来源的知识页。
3. 系统检查这个知识页是否可靠。
4. 之后类似问题优先命中这条整理过的知识。
5. 知识页之间还可以建立关联，比如  `related_to` 、 `supports` 、 `refines` 、 `conflicts_with` 。

这样用久了以后，它不是只有一堆原始帖子，而是会慢慢形成一层更好用的知识库。

## 4. 和普通 RAG 的区别

普通 RAG 很多时候是“检索一批 chunk，然后让模型临场组织答案”。但是这种思路有两个问题：

首先，答案质量很依赖当次检索。如果这次没搜到关键帖子，回答就会偏。

其次，模型每次都要重新理解同一批资料。明明之前已经总结过一次，下次还是从头再来。

这个系统想解决的是第二个问题。搜索只是入口，真正重要的是把高质量结论沉淀下来，让下一次使用变得更轻。

改进后的系统：原始资料保留，整理后的知识也保留；需要严谨的时候可以回到原帖，需要效率的时候可以直接用知识页。

## 5. 当前已经完成的部分

现在已经完成的功能包括：

- 论坛 JSON 解析和 SQLite 本地索引。
- 本地搜索、精确查找、查看原帖、相关帖子推荐。
- MCP server，给 AI Agent 调用。
- 自进化知识页：创建、搜索、查看、发布、lint。
- 知识页来源绑定，避免没有证据的总结混进知识库。
- 知识页之间的关系图谱。
- Markdown Wiki 导出，方便人工检查。
- README 和专门给 Agent 看的 README_AGENT。

整个系统默认不依赖外部 LLM API，也不需要外部向量数据库。会用 MCP 的人拿到代码以后，基本就是  `uv sync` ，然后配置一下 MCP，就可以让 Agent 开始使用。

## 6. 适合什么场景

- 有大量历史讨论、文档、经验贴。
- 搜索本身很重要，但搜索结果还需要判断。
- 很多问题会反复出现，不想每次都重新找资料。
- 希望 Agent 不只是回答问题，还能把有价值的经验沉淀下来。

对我来说，最直接的用途就是 alpha 挖掘时查论坛经验、查 operator 用法、查历史讨论、查一些容易被遗忘的细节。

目前这个版本我觉得已经可以作为一个比较轻量的本地 Agent 知识系统来用了。它不追求一开始就很复杂，而是先把“能搜、能查证、能沉淀、能复用”这条闭环跑通。
