# WQ BRAIN 表达式验证与性能优化综合指南

在 WorldQuant BRAIN 平台上开发高质量、低相关的 Alpha 表达式时，盲目的试错和参数微调往往会消耗大量请求次数。为了实现“零预算”或极高效率的策略开发，我们需要将**本地表达式严格校验 (Validation)** 与**系统化、结构化的性能优化机制 (Optimization)** 深度结合。

本文档将工具 `validate_expression` 的前置校验能力与工作流 `WQ_BRAIN_ALPHA_OPTIMIZATION_V1` 中的性能提升提示词规则融为一体，为您提供一份从生成、验证到回测、降相关的端到端综合使用说明。

## 🛠 一、 核心底座：本地表达式验证工具

在任何候选策略提交给平台进行回测（如 `create_multiSim`）之前，**必须**使用本地工具 `validate_expression` 进行预检。该工具扮演了“Fail Fast（快速失败）”的守门员角色。

### 2. 在工作流中的定位

- 生成的每一条候选 Alpha 均要执行该工具。

- 若 `is_valid` 返回 `False`，必须根据返回的 `errors` 进行**逐条精确修正**。

- **严禁降级妥协**：当发现复杂的表达式提示验证失败时，只能针对报错点修复语法和参数关系，禁止通过删减核心算子或为了妥协而替换为简单无效的算子。

## 🚀 二、 策略生成与性能提升指南 (Improve Theme V1)

在确保每一条被提交的表达式在**代码语法上绝对正确**之后，我们需要在**金融逻辑提取和指标表现**上进行结构化提升。

**此处不严格限制表达式模版的使用，以最大自由度发挥 AI 的能力。**

### 1. 目标与完成标准 (Definition of Done)

我们要基于当前的基准 Alpha（锁定核心数据字段，不随意更换），使用大模型生成新的 8 条变体表达式阵列，直至产出的 Alpha 满足：

- **收益指标**：Sharpe > 1.58，Fitness > 1.0，Turnover 在 1%-40% 之间。

- **独特性**：与平台生产库 Alpha 相关性 (PROD correlation) 严格 `< 0.7`。

### 2. “反同质化” 的组装配额

为了避免生成的 8 条表达式陷入简单的参数微调，系统制定了严苛的组装配额：

- **Stage A（探索期，指标未达标）**：禁止**一切纯参数微调**（Fine-Tune）。每次迭代必须引入或替换新的核心数据结构算法。例如，引用特定的条件冻结 (`trade_when`)、截断去噪 (`winsorize`, `hump_decay`) 或正交去共线 (`group_vector_neut`)。

- **强制主题覆盖**：每一批的 8 条策略必须覆盖至少 4 种不同的算子主题类别，避免将策略逻辑单一化。

- **常用算子限制**：为了倒逼探索不常用的有效操作符（如 `generalized_rank`），对于 `rank`、`zscore` 等高频算子，同一批中针对特定操作符的重复使用受到了严格次数限制。

### 3. 反向指标的“废物利用” (Short Flip)

系统会对回测表现极度糟糕的表达式（如 Sharpe ≤ -1.20 且 Fitness ≤ -0.50）进行敏锐捕捉。这些通常是极强的反向信号。

- 这些策略会被打上 `CAND_NEG` 标签。

- 优化流程在下一轮迭代时，必须强制生成至少两项它的**反向翻转版本**（例如包在 `multiply(-1, ...)` 中或最外层取反），化腐朽为神奇。

## ⚙️ 三、 综合实战流水线总结

从接收到原始的探索灵感，到最终产生一个达标（Done）的 Alpha，严格遵循以下闭环：

1. **确定基线与约束**：利用 `authenticate` 与 `get_alpha_details` 获取并冻结当前试图优化的核心数据字段和目标市场（Region/Universe）。

2. **构思与生成 8 条阵列**：AI 根据分配的角色（探索稀有算子或是修复特定噪声）书写 8 种变幻版本的结构，并检查没有踩到常用算子的配额红线。

3. **AST 本地审判 (Preflight & Validator)**：

利用 `validate_expression` 进行扫描：

```python

is_valid, errors, b_fields = validate_expression(expr, csv_path="...", region="...")

```

*凡是报错的（如缺失参数名、算子错误），原地重写，不得进入下一步。*

4. **云端多轨回测 (`create_multiSim`)**：8 条完美语法的表达式将零语法错误地推进平台模拟。

5. **诊断判决**：

- 检查谁跑出了负分高表现？引入下一轮反转计划。
