## 帖子正文

##### ts_target_tvr_decay(x, lambda_min=0, lambda_max=1, target_tvr=0.1)

需要输入的参数为以上4个。

通过lambda_min值（不衰减），和lamdba_max值（衰减）的搜索空间中，去搜索lambda（λ)值，来得到target_tvr值。

也就是说，足够大的搜索空间，通过能得到一个λ，使得alpha的tvr为target_tvr值。

而目前多数顾问采用的是

##### lambda_min=0, lambda_max=1, target_tvr=[0.1,0.15,0.2,0.25] 这样的用法。

但按照我个人理解，设置成

##### lambda_min=0, lambda_max=5, target_tvr=[0.01，0.15，1]   （其中1是tvr参数的是上限值）

在扩大搜索空间后，使得alpha tvr的变化呈现缩小值、理想值、强化值。 这样的用法会更有意义。

**因为与其在固定的搜索空间去趋近target_tvr ，不如扩大搜索空间，跳出局部求解更为实际。如果将**

##### **lambda_min=0, lambda_max=5, target_tvr=0.15作为模板，那么搜索对于tvr弱的alpha，λ值会趋向于5，旧信号衰减更快，增强tvr,趋近0.15。对于tvr强的alpha，λ值会趋向于0，旧信号衰减更慢，降低tvr趋近0.15。是一个自适应的模板。类似decay的用法。**

若如果不存在相应的λ值，0.01和1也使得tvr的变化朝减弱和增强进行了延伸。综上所述，tvr 操作符总是能将tvr值按照自己理想的方向进行变化，是一个很实用的操作符。

比如这个alpha，看到tvr为2.58%， 通常是认为是一个beta而放弃提交。

### 评论 #20 (作者: AM12075, 时间: 9个月前)

tvr 的优化依赖于历史数据，如果历史波动或极端事件过多，搜索出的 λ 可能偏向极端值，导致未来实际信号不稳定。扩大 lambda_max 到 5 会极大增加衰减范围，对历史信号依赖强的 alpha 可能导致过度衰减，损失有用信息。其次target_tvr 设置极端值（如 0.01 或 1）可能导致 λ 计算不收敛或极端化，使 alpha 行为异常，尤其在小样本或低流动性股票上。

"其次target_tvr 设置极端值（如 0.01 或 1）可能导致 λ 计算不收敛或极端化，使 alpha 行为异常，尤其在小样本或低流动性股票上。"

使用极端值的情况，是因为tvr正常设置0.15时 无法达到理想的区间如0.125-0.175。 这应该本身就对应了不收敛的情况。说明了alpha自身就是一个低频/高频信号。因此通过极端值去扭转了频率，逼近0.15来达到alpha在换手率上的可用性。如果后续的os部分引入的历史数据相差较大，导致了alpha频率发生了较大变化，影响了λ和target_tvr。那可能是其他的数据处理方式也难以避免的。

这是一个非常敏锐的观察。直觉上，增加“衰减”（Decay）或“平滑”应该总是让信号变慢、换手率（TVR）降低，但在某些特定情况下，ts_target_tvr_decay 确实可能导致 TVR 不降反升，或者比你预期的“平滑效果”产生更大的波动。这通常由以下三个深层机制导致：1. 均值回归信号的“趋势化”陷阱如果你的原始信号 $x$ 是一个高频震荡或强均值回归的信号（比如在 0 附近快速跳变）：原始状态： 信号不断穿过零点，正负抵消，如果你不持仓或者持仓极小，换手率可能并不高。衰减后： ts_decay 会把这种高频跳变累积成一个有方向的短趋势。原本快进快出的微小跳动，变成了具有一定动量的持续信号。结果： 这种“被制造出来的趋势”可能导致你的调仓动作更加连续且幅度更大，从而在计算 TVR 时发现它比原始信号更“活跃”。2. 优化器的“边界超调”（Overshooting）ts_target_tvr_decay 是一个在 lambda_min 和 lambda_max 之间寻找平衡的求解器。机制： 如果你的 target_tvr 设置得比原始信号本身的 TVR 还要高，优化器为了“达标”，会倾向于向 lambda_max（通常是 1，即不衰减）靠拢。不稳定性： 在迭代过程中，如果信号本身在截面上分布极不均匀（个别股票波动极大），优化器为了让整体组合达到平均 0.1 的 TVR，可能会给某些平滑的股票分配更“激进”的参数，导致局部 TVR 激增。3. 信号的“量纲放大”效应这是最常见的原因。衰减操作符往往会改变信号的统计分布（如标准差）：原始信号： 可能是离散的、有很多噪声，很多值接近 0。衰减后： 噪声被平滑，信号变得更加“平滑且丰满”。如果后续接了 zscore 或 rank，这些原本微弱的信号被拉伸到了整个坐标系中。结果： 虽然变化频率变慢了，但每次变化的**幅度（Magnitude）**变大了。TVR 的计算公式是 $\sum |w_t - w_{t-1}|$，如果单次变动幅度显著增加，即便变动频率降低，总和依然可能变大。4. 滞后导致的“追涨杀跌”所有的 Decay 都会带来相位滞后（Phase Lag）。当原始信号已经见顶回落时，Decay 后的信号可能还在惯性冲高。这种滞后会导致在信号反转期，Decay 后的信号产生的调仓指令与市场步调不一致，甚至在短时间内产生剧烈的仓位修正。

### 评论 #34 (作者: PP16638, 时间: 5个月前)

学习了文档里这一大把 decay 算符, 目前的理解是:

`ts_decay_linear(x, d, dense = false)`:  **在过去d天内线性衰减** 

按照文档例子, 假设某值 x 在过去五天内 当天=30 (outlier); 1天前=5; 2天前=4; 3天前=5; 4天前=6
这个例子里, 当天是异常值 (30), 而昨天, 前天等等数值还算正常
则 `ts_decay_linear(x, 5)` 是 (30⋅5)+(5⋅4)+(4⋅3)+(5⋅2)+(6⋅1) / (5+4+3+2+1) = 13.2 
可见削弱了当天的异常值影响
其中, dense = false/true 涉及如何处理 NaN 值, dense = false 时 NaN 视为零, 这个先略过

`ts_weighted_decay(x, k=0.5)` :  **在今值和前值之间线性混合** 

我没有 `ts_decay` 算符, 根据文档的线索, `ts_decay(x, 1)` 是完全放弃今值, 只取前值

而 ts_weighted_decay 可视为精细版本的 `ts_decay(x, 1)` 用于在今值和前值之间任意比例混合
还是文档的例子, 假设今天是 6 昨天是 4, 则 `ts_weighted_decay(x,k=0.25) = 6*0.25+4*0.75 = 4.5` 
该算符没规定 k 范围, 看起来 k 合理范围是 [0,1], k 是指今天的权重

现在, 终于轮到
`ts_target_tvr_decay(x, lambda_min=0, lambda_max=1, target_tvr=0.1)` : 文档解释是

> Tune "ts_decay" to have a turnover equal to a certain target, with optimization weight range between lambda_min, lambda_max

个人理解是:  **一种加强版本的 ts_decay, 在 lambda_min, lambda_max 之间搜索一个 lambda, 使得 turnover 尽量靠近 target_tvr,** 它像是个 "自动版本" 的 `ts_weighted_decay(x, k=0.5)`, 在内部以 lambda_min, lambda_max, target_tvr 共同决定出来 k 应该是多少
那么看起来 lambda_min=0, lambda_max=1 是合理的, 但它没规定 lambda 必须小于 1, 嗯, 感谢楼主分享, 我再试试
