I hit a new frustration today. I guess there is a lockdown on simulating if you have too many alphas in a particular category. Here's what I got:

> Overused data error "fnd6_xxxxx" field(s) in "Fundamental".

A couple questions here -- (1) does this only apply to Fundamental, or do other categories also hit this limitation? (2) submission I understand, but why block simulation? It kinda throws a wrench in the data exploration process in the background. (3) I am in the gold level jail, but it seems these percentages are still including all the data categories I don't have access to. This creates a situation where everything goes red simply because it is not possible to submit alphas in all those other categories. This seems like a BUG with the GENIUS levels. (4) The docs say one needs to get below 15% to get out of the jail, however after the 1 fundamental alpha I submitted today I am over 30% fundamental and that's when I experienced the lockdown. So, are the docs correct about 15%, or is it 30%?

Help, please! (no AI responses!) I'm starting to go a bit nuts with all the restrictions.

### 评论 #3 (作者: AC63290, 时间: 1年前)

It sounds like you're dealing with a particularly frustrating aspect of the GENIUS program's data usage restrictions. Let me address your questions one by one:

1️⃣  **Does this only apply to Fundamental, or do other categories also hit this limitation?** 
Restrictions typically apply across all data categories, not just Fundamental. The limitations are in place to ensure diversity in Alpha submissions and avoid overuse of any single data category. You’ll want to explore alternative datasets to reduce reliance on Fundamental data.

2️⃣  **Why block simulation?** 
Blocking simulations seems counterproductive for data exploration. It could be a measure to enforce balance across categories, but I agree it hinders the creative process. You might want to raise this as feedback directly to the team managing the program.

3️⃣  **Percentages including inaccessible categories:** 
This does seem like a bug. If you're being penalized for categories you can’t access, it skews your percentages unfairly. Document this issue and escalate it to the support team for clarification and resolution.

4️⃣  **15% vs. 30% Threshold:** 
Based on your experience, it sounds like the 15% rule might apply to jail exit conditions, but a stricter limit (30%) triggers simulation restrictions. Check if there’s been an update to the documentation or if this discrepancy is an oversight.

This level of restriction can feel stifling, especially when you're trying to innovate. Hopefully, these clarifications help, and escalating the potential bugs will lead to improvements in the program. Hang in there!

Hello! You need to make more alphas using non-fundamental data in the region where you get "over-used data test fail". This restriction will automatically be removed once the contribution of your fundamental data alphas goes below 15%.

You can read more about it here:

[https://support.worldquantbrain.com/hc/en-us/articles/22696581922071-How-can-consultants-regain-access-to-the-Fundamental-dataset-category](/hc/en-us/articles/22696581922071-How-can-consultants-regain-access-to-the-Fundamental-dataset-category)

There are some datasets in analyst, and model data categories that have some more commonly used fundamental data. So those can be a workaround if you have to use fundamental data.
