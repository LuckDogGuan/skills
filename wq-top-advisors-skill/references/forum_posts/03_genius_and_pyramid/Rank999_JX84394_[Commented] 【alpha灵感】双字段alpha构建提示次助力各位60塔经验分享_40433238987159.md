你现在这4类已经覆盖了“双字段 regular alpha”的核心骨架，但如果目标是做一个真正全面、可系统化枚举/研究/模板化的框架，我建议你扩展成下面这 12 大类。

而且这些类别最好不是“表达式层”分类，而是：

- 信号关系层（经济含义）
- 结构层（拓扑）
- 角色层（主次）
- 时间层（先后）
- 横截面层（分组）
- 状态层（条件）
- 统计层（联合分布）

这样 AI 后面才能真正做：

- 模板学习
- topology diversification
- hypothesis-driven generation
- failure attribution

而不是只会“字段A op 字段B”。

# 一、字段关系型（你已有，但需要细化）

这是最基础层。

## 1. 比例关系（Relative Scale）

例如：

```
A / B

```

本质：

- 单位化
- 强弱相对化
- 结构占比

常见：

- valuation
- leverage
- efficiency
- utilization

例子：

```
earnings / assets

```

## 2. 差值关系（Spread）

```
A - B

```

本质：

- 偏离
- 错配
- gap

非常重要。

很多 alpha 本质都是 spread。

例如：

- analyst vs realized
- short vs long
- sentiment vs price

## 3. 差分关系（Relative Change）

不是简单 delta。

而是：

```
ts_delta(A) - ts_delta(B)

```

或者：

```
delta(A/B)

```

本质：

- 谁变化更快
- acceleration mismatch

这是很多低相关 alpha 来源。

## 4. 求和/组合（Composite Exposure）

```
A + B

```

注意：

不是简单“混信号”。

而是：

- 同方向不同维度
- 同一 latent factor 的 proxy 聚合

否则容易破坏 atom purity。

## 5. 对数/非线性映射

例如：

```
log(A/B)

```

```
sqrt(A*B)

```

本质：

- 压缩尾部
- 放大中间结构
- 稳定分布

# 二、主信号 + 辅助信号（你已有，但还可以细化）

这个极其重要。

很多 regular alpha 不是“双主信号”。

而是：

```
main signal
+
context signal

```

## 6. Confirmation（确认型）

辅助字段用于确认主字段。

例如：

```
trade_when(
    sentiment > 0,
    alpha_main,
    -1
)

```

本质：

- 降低噪音
- 提高 hit-rate

不是增强 alpha 强度。

## 7. Gating（门控型）

辅助字段决定：
“什么时候允许主信号生效”。

例如：

```
if(volume > threshold)

```

这是非常重要的一类。

很多 alpha 本身没问题，
只是：

- 在错误 regime 下失效。

## 8. Risk Filter（风险过滤型）

辅助字段用于：
去掉高风险状态。

例如：

- volatility
- liquidity
- crowding
- beta

这是 production alpha 非常重要的一层。

## 9. Regime Switch（状态切换型）

不同市场状态下，
同一个主信号方向不同。

例如：

```
if(vix_high)
    reversal
else
    momentum

```

这是高级 alpha 常见结构。

# 三、分组/横截面结构（你已有，但不够完整）

## 10. 自定义分组（Bucket Group）

你已经提到：

```
bucket(field_B)

```

这个非常关键。

本质：

字段A在字段B定义的“经济层级”中的异常。

例如：

- valuation within leverage bucket
- sentiment within liquidity bucket

这是低相关 alpha 非常重要来源。

## 11. 双字段联合分组（Joint Group）

这个很多人没系统化。

例如：

```
bucket(A) × bucket(B)

```

形成二维 regime。

比如：

- 高波动+高拥挤
- 低估值+高成长

这是高级横截面结构。

## 12. 动态分组（Adaptive Group）

不是固定 sector。

而是：

```
group = bucket(ts_rank(B))

```

即：
分组本身会动态变化。

这非常适合：

- sentiment
- crowding
- volatility

# 四、双字段操作符（你已有，但建议独立成体系）

## 13. 统计关系型操作符

例如：

- correlation(A,B)
- covariance
- regression residual
- beta
- cointegration-like

本质：

不是字段值本身，
而是：
字段之间关系。

这类 alpha 很高级。

## 14. 排序关系型

例如：

```
rank(A) - rank(B)

```

或者：

```
ts_rank(A) - ts_rank(B)

```

本质：

- 相对位置错配

很容易出稳定 alpha。

## 15. 分布偏离型

例如：

```
zscore(A within B-group)

```

或者：

```
A - mean(A | B-bucket)

```

本质：

- conditional anomaly

这个非常适合做 regular alpha。

# 五、时间结构（很多人漏掉）

## 16. Lead-Lag（领先滞后）

极其重要。

例如：

```
A_t vs B_{t-k}

```

本质：

- 信息传播速度
- delayed reaction

很多 alpha 都来自这里。

## 17. Horizon Spread（期限结构）

例如：

```
ts_mean(A,5) - ts_mean(B,60)

```

或者：

```
short-term vs long-term

```

本质：

- term structure
- persistence mismatch

## 18. Persistence vs Shock

例如：

```
persistent(A)
vs
spike(B)

```

这是 news/sentiment 特别重要的一类。

# 六、角色型（非常容易被忽略）

这是 RD-Agent 风格里特别重要的一层。

不是“字段是什么”。

而是：

字段扮演什么角色。

## 19. Anchor + Residual

例如：

```
A - expected(A|B)

```

B 是 anchor。

真正交易的是 residual。

这是高级 alpha 核心思想。

## 20. Driver + Reaction

例如：

- field A 是原因
- field B 是市场反应

交易：
“反应不足/过度”。

这个很适合：

- analyst
- news
- options
- crowding

## 21. State + Action

例如：

- B 定义状态
- A 定义动作

比如：

```
if(crowded)
    reversal
else
    continuation

```

# 七、信息论/异常型（高级扩展）

## 22. 一致性/背离型（Consistency Divergence）

例如：

```
sign(A) != sign(B)

```

或者：

```
high A but low B

```

本质：

- 信息冲突
- 市场未定价

## 23. 稀缺共振型（Rare Co-occurrence）

例如：

两个罕见事件同时发生。

这类 alpha：
coverage 低，
但相关性极低。

## 24. 条件异常型（Conditional Anomaly）

不是：

```
A abnormal

```

而是：

```
A abnormal given B

```

这是最重要的一类之一。

真正高级 alpha 大部分属于这里。

# 八、建议你最终形成“二维 taxonomy”

不要只列模板。

应该形成：

维度
内容

关系类型
ratio/spread/residual/lag

角色类型
main/filter/gate/regime

时间结构
short-long/lead-lag

横截面
bucket/joint-group

统计结构
corr/residual/zscore

异常类型
conditional anomaly

topology
TS→CS / CS→TS

signal shape
level/change/persistence

这样 AI 才能真正：

- 做 topology diversification
- 做 hypothesis coverage
- 避免只会生成同构 alpha

这和你之前强调的：
“不是暴力生成，而是研究型 workflow”
是完全一致的。

你下一步最值得做的，其实是：

# 建立“双字段模板 AST”

例如：

```
template:
  relation_type: spread
  role_type: anchor_residual
  topology: TS->CS
  conditioning: bucket_B
  signal_shape: anomaly

```

然后 AI 只能：

- 在合法 topology 内变异
- 在合法角色内替换
- 在合法 relation 内扩展

这才会真正接近 Microsoft Research 的 RD-Agent 风格研究系统，而不是普通 alpha generator。
