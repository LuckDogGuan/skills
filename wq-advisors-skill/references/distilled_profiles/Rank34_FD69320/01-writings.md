### 普通随机爬山算法的基本结构

1. **初始化** ：从初始参数开始。
2. **评估** ：回测表达式并获得其适应度。
3. **选择** ：通过随机选择一个不同的参数来识别邻近组合。
4. **比较** ：重新回测更新后的表达式并获得其适应度。
5. **更新** ：如果邻近表达式显示出更好的适应度，则将当前选择更新为这个新参数集。
6. **迭代** ：重复评估、选择、比较和更新步骤，直到在n次迭代后没有进一步的改进。
7. **重启** ：重复上述步骤m次。这是重启机制。

### 智能重启 🔄

要考虑的第一个修改是确保在第七步的每次重启中，算法都会探索以前未遇到的新区域。为实现这一点，您可以在步骤三和四中实现一个参数计数器，每次选择并模拟一个选项时计数器递增。在第七步重启时，不要随机从可用集合中抽取选项，而是使用计数器的逆作为选择的概率。此修改确保每次重启都会选择在先前试验中未探索过的参数。🧠

### 直线路径可能不是最佳路径 🌄

正如一些人指出的那样，随机爬山算法的贪婪特性使其过于关注即时改进，从而可能忽略更优的曲线路径。有些人建议使用模拟退火作为替代，因为它在探索期间会容忍挫折，这有助于达到全局最优。为了使随机爬山算法达到类似的效果，您可以在步骤四中引入一些白噪声。这增加了选择“最佳”邻居的随机性。

有时，“最佳”邻居的适应度可能在短期内不佳，但在未来步骤中有更好的解决方案潜力！您还可以设计动态噪声，在早期阶段引入更多白噪声，并在后期逐渐减少——类似于模拟退火中的温度衰减。❄️🔥

### 一.原有痛点

### 二.解决方案

使用数据库,本方案采用的结构化数据库MySQL,也有同学使用的Mongo等非结构化的,这个我抛砖引玉,欢迎讨论

我这边独特的字段如下

1.tags,自己打标签.multi_simulate的时候,可以获取到progress_urls,把这些url和应该打的标签在代码中存起来推到redis中,另起一个任务专门负责根据这些url获取其alpha的id,进而和tag建立关系,再转存到MySQL中

2.check_status.接口获取信息到的check信息中,存在fail的,这个字段就为fail,否则就是pending,只有pending的alpha才需要后面check

3.low_2y_sharpe.这个字段是single的dataset独有,能获取到就赋值,获取不到就补1.不是补0,因为为0的需要将check_status变为fail

4.is_ladder_sharpe,不是single的dataset,则有这个字段,同样的,能获取到就赋值,获取不到就补1.不是补0,因为为0的需要将check_status变为fail

5.is_submit.记录是否提交的信息.0未提交,1已提交.值得注意的是,这个字段还能解决如何科学存货.按自己的提交标准,check完以后把is_submit字段赋值为2-10之间的数值.比如我,2为优秀10为垃圾,中间酌情.

def is_dataset_submitted(        self,         dataset_id: str,        region: str,        universe: str,        delay: int,    ):    """    Check if a dataset has been submitted.    Args:        dataset_id (str):   The dataset ID to check.        region (str):       The region of the dataset.        universe (str):     The universe of the dataset.        delay (int):        The delay of the dataset.    Returns:        bool: True if submitted, False otherwise.    """    result = True    selection = BRAIN_DATASET_SUBMITTED_SELECTION.format(dataset_id=dataset_id)    alphas = self.super_alpha_selection(selection, region=region)    if alphas.empty:        result = False    else:        alphas['region'] = alphas['settings'].apply(lambda x: x['region'])        alphas['delay'] = alphas['settings'].apply(lambda x: x['delay'])        alphas['universe'] = alphas['settings'].apply(lambda x: x['universe'])        if alphas.loc[(alphas['region'] == region) & (alphas['delay'] == delay) & (alphas['universe'] == universe)].empty:            result = False    return result

**问题：**

得益于之前 [MH33574](/hc/en-us/profiles/25669350433175-MH33574) 的启发  在过去一段时间中，我会激进的选择三个不同类型的alpha作为一阶去批量回测，分别是单一数据集，推荐字段，以及论坛模板。三个类型分别占用三个卡槽做回测，留一个卡槽自己用。然而，回测一段时间后想要从跑完的结果中找出合适的alpha跑二阶时就会发现不同的类型生成二阶的方式都不一样。

**如何解决：**

## 1. ts_zscore

***功能说明*** ：计算时间序列的Z-score标准化值。
 ***参数说明*** ：

- - x: 输入数据
- - d: 回看天数

Z-score定义：Z-score(标准分)是一个统计学概念，表示一个观测值与平均值的偏离程度，以标准差为单位。

Z-score = (原始值 - 均值) / 标准差

假设我们有一只股票最近5天的收盘价：100,102,101,103,99

1. 均值 = (100 + 102 + 101 + 103 + 99) / 5 = 101元
2. 差值平方：(100-101)² + (102-101)² + (101-101)² + (103-101)² + (99-101)²
       = 1 + 1 + 0 + 4 + 4 = 10
3. 标准差 = √(10/5) ≈ 1.414元

- z-score为负表示低于平均水平
- z-score为正表示高于平均水平
- |z-score| 越大表示偏离程度越大

***使用示例：***

```
# 基于均值回归的反转策略-ts_zscore(returns, 5)  # 短期超涨/超跌反转# 均值回归策略-ts_zscore(close/ts_delay(close,1) - 1, 5)  # 短期收益率均值回归
```

## 9. hump

***功能说明*** ：限制输入值的变化频率和幅度，从而减少换手率（turnover）。

***工作原理：***

- - 如果今天的值相比昨天的变化很小（不超过阈值hump），输出保持与昨天相同
- - 如果变化超过阈值，输出为昨天的值加上或减去阈值（根据变化方向）

> 今天的值 vs 昨天的值
> ↓
> 变化是否超过阈值hump？
> ↓
> 是 → 昨天的值 ± hump（根据变化方向）
> 否 → 保持昨天的值

***使用示例：***

```
# 示例1：限制alpha信号的变化hump(alpha_signal, 0.01)  # 限制信号变化在1%以内# 示例2：结合trade_when使用trade_when(    volume > ts_mean(volume, 20),    hump(alpha_signal, 0.02),  # 高成交量时允许2%的变化    hump(alpha_signal, 0.005)  # 低成交量时只允许0.5%的变化)
```

## 1. ts_zscore

***功能说明*** ：计算时间序列的Z-score标准化值。
 ***参数说明*** ：

- - x: 输入数据
- - d: 回看天数

Z-score定义：Z-score(标准分)是一个统计学概念，表示一个观测值与平均值的偏离程度，以标准差为单位。

Z-score = (原始值 - 均值) / 标准差

假设我们有一只股票最近5天的收盘价：100,102,101,103,99

1. 均值 = (100 + 102 + 101 + 103 + 99) / 5 = 101元
2. 差值平方：(100-101)² + (102-101)² + (101-101)² + (103-101)² + (99-101)²
       = 1 + 1 + 0 + 4 + 4 = 10
3. 标准差 = √(10/5) ≈ 1.414元

- z-score为负表示低于平均水平
- z-score为正表示高于平均水平
- |z-score| 越大表示偏离程度越大

***使用示例：***

```
# 基于均值回归的反转策略-ts_zscore(returns, 5)  # 短期超涨/超跌反转# 均值回归策略-ts_zscore(close/ts_delay(close,1) - 1, 5)  # 短期收益率均值回归
```

## 9. hump

***功能说明*** ：限制输入值的变化频率和幅度，从而减少换手率（turnover）。

***工作原理：***

- - 如果今天的值相比昨天的变化很小（不超过阈值hump），输出保持与昨天相同
- - 如果变化超过阈值，输出为昨天的值加上或减去阈值（根据变化方向）

> 今天的值 vs 昨天的值
> ↓
> 变化是否超过阈值hump？
> ↓
> 是 → 昨天的值 ± hump（根据变化方向）
> 否 → 保持昨天的值

***使用示例：***

```

## 12. ts_decay_exp_window

***功能说明：*** 对时间序列进行指数衰减加权平均，近期数据获得更大权重。

***参数说明：***

- - x: 输入序列（必需）需要进行衰减处理的原始数据
- - d: 时间窗口（必需）回溯的天数
- - factor: 平滑因子（可选） 控制衰减速度(0, 1)
- - 越接近0衰减越快
- - 越接近1衰减越慢
- - nan: NaN处理方式（可选）
- - True：如果输入值为NaN则返回NaN
- - False：如果所有值都是NaN则返回0

***工作原理：***

```

## 49. abs

***功能说明：*** 返回输入值的绝对值。

## 50. sign
