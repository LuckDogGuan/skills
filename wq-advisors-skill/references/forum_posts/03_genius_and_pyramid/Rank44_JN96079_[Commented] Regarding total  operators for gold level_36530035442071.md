### 评论 #6 (作者: TL60820, 时间: 6个月前)

You are conflating syntax availability with resource allocation. The library of mathematical operators (e.g., ts_decay, group_rank) is standardized across the platform; a Gold consultant uses the same functional dictionary as a Master. There is no "locked" set of operators that unlocks at higher tiers.

The actual differentiation lies in simulation budgets (concurrent backtests), submission quotas, and dataset access. Gold status typically grants a higher ceiling for active alphas in production and arguably faster simulation queues, but it does not change the operator syntax itself.

You’re mixing up operator access with resource limits. The mathematical operator library—such as ts_decay or group_rank—is the same for Gold and Master users, with no functions locked behind higher tiers. Gold users generally get more backtesting bandwidth and faster queues, but the underlying operator syntax remains identical across tiers.
