## 帖子正文

## 1.1  **楔子**

数学的本质是对现实世界的精准刻画。而金融市场也是现实世界的组成部分之一。我们研究的量化因子，如果以物理学的视角，金融市场就是我们研究对象。我们每天寻找的因子，则是在试图精确地刻画市场中风险和回报的关系，作为我们通过市场低效率获利的一个着力点。

同时，这种通过“低效率”获利不是永恒的。一个有效的因子被发现和使用的人多了，它带来的超额收益就会逐渐衰减。因为随着交易量的无上限增加，必定会达到影响市场的走势的数量级。这时，市场重新变得平滑，我们找到的因子也因此完成它的历史使命。如果把这个过程看过一个物理运动的过程，从现实的角度，我们必须不断迭代因子，寻找新的“着力点”。所以，我们必须做一个“因子猎人”，这远比做一个“金矿的矿主”更有挑战性和吸引力。

## 1.2  **两个事实**

第一个基本的事实，构建有经济学意义的模板可以大大提升我们的因子输出效率，而因子模板客观地存在适用性。

有的模板特别适合量价数据集，

例如

```
<time_series_op/>(<profit_field>/<size_field>, <days>)
```

特别适合量价数据集带有profit和size的数据字段

再如

```
<group_op>(<put_greek> - <call_greek>, <grouping_data>)
```

特别适合期货数据集，1000次回测出200个信号非常丝滑，非常适合新人加强自信。

再如有些模板在特定的区域很有效，在有些区域的因子输出效率很低。这些模板我想称之为定制模板，因为他们在某一个Region、Category或包含某1-2个关键词的时候因子输出效率很高。

有没有可以自由泛化的，可以适应所有Region、所有Category的标准化模版？

有的，我们很多人在用户阶段常用的模板

```
Group_op(ts_op(datafield, day), group)
```

就是一个可以自由扩展到所有Region和Category的模板。这个模板在控制了某个共同特征（如行业、市值）的前提下，寻找那些在时间序列上表现出特定模式（动量或反转），并且这种表现在同类资产中具有相对优势的股票。我把这种模板成为标准模板，这也是我想重点讨论的模板。

第二个基本事实，很多的标准模板已经被全球的前辈跑的七七八八了，能交的也交的差不多了。传统的标准模板对我们新顾问而言，属于可望而不可求的“鸡肋”。所以我们必须具备模板构建的能力，而且必须构建高产出模板的能力。

如何构建模板，现实中有很多个解决方案。通过MCP让LLM生成（贵，有点费米）；把已提交的模板进行运算符的泛化，数据字段的泛化（模板框架有限，泛化有数量上限）；阅读专业文献，从中寻找灵感（专业词汇很多，学习曲线相对陡峭，有一定的学习成本）；以及手搓（主要靠自身的经济学和数学认知，除非自己懒，一般没有上限）。在这几个解决方案里面，最有成就感的是手搓，而且PC相对最低，相对容易把控。

## 1.3  **手搓模板三部曲**

我认为，标准模板的本质是一种数学逻辑对市场风险的精准刻画，而Alpha因子是这种数学逻辑的具体表现。根据模板含有唯一数据字段的个数，我把标准模板分为一元模板、二元模板和三元模板。而手搓模板就是结合自己的数学和经济学认知，把自己的金融直觉，用数学的方式表达出来。

手搓一个模板可以分成三个大步骤。

第一，数据基础处理

主要包括填补，去极值，除数不为零，平均值，标准差，最大值最小值，绝对值，降维等操作。理由就很简单，dirty的数据没有数学意义，更没有经济学意义。噪音大了，啥都听不清。 

第二、构建比较关系

在用户阶段的时候，我认真看了论坛上置顶的模板和模板构建的策略，当时有个很模糊的感觉：很多的因子模板的底层运算逻辑都是在构造某个不同或比较，可以是单个数据字段，也可以是两个数据字段，或者三个或者更多，通过“某种比较”来刻画某个差异性，构造一组向量，最终通过这组向量来实现中性化交易。运算符的作用则是对这种差异化进行定制和深化，使得这种不同具备金融学的现实意义。所以，我觉得构建模板的核心应该是构建某个比较关系。

要完成这个步骤，我觉得可以分成两个小步骤，
步骤一，逻辑运算

例如，设置触发条件

```
trade_when()
```

```
if_else()
```

这个步骤，在构建简单的模板是非必需的，但我觉得这是一个高级动作。类似代码的控制流if 和while的结合体。这个步骤可以根据自己的想法决定是否需要加入。

步骤二，主运算符构建比较

通常包括 减法，除法，排名，打分等，也可以根据实际情况构建高维度差异。下面举几个例子。

例1 差值组合（这里省略第一个步骤数据基础处理环节，下同）

```
ts_delta(datafield_1, day)
```

例2 减法和排名的组合

```
Rank_op(datafield_1- datafield_2)
```

例3 除法和打分的组合

```
Zcore_op(datafield_1/datafield_2)
```

例4 线性回归组合

```
ts_regression(datafield_1, datafield_2, day, rettype = 0)
```

同时，我们也可以把这些基础因子用乘法做连接，构建多元因子模板。但多元模板跑起来很慢，除非产出可以，一般不推荐新顾问使用。

然后，很重要的一步，经济学意义检验

把这个模板，在一个category找一个数据字段试跑一下，看有没有信号？同时，请LLM看一下，有没有经济学意义？光有数学意义，没有经济学意义的模板大概率产出效率很低。毕竟，我们的核心是对各种风险因素进行刻画，唯一的检验标准是市场，是数据，是盈利，不是我们的神光一现。

第三、细节调整
如果有信号，而且有经济学意义，那就进行第三阶段，加上辅助优化运算符。例如分组，幂、对数等方式，强化放大信号。在可以做空的市场，是否需要加上负号 。

最后，把所有通过经济学意义检测，有信号的模板，建立模板群，按照一元、二元和三元模板分别随机批量回测。市场随机因素很多，我们就以随机的模式应对市场。
