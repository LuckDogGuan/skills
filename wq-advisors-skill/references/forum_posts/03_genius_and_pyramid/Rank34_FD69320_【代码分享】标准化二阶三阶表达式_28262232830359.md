## 帖子正文

**问题：**

得益于之前 [MH33574](/hc/en-us/profiles/25669350433175-MH33574) 的启发  在过去一段时间中，我会激进的选择三个不同类型的alpha作为一阶去批量回测，分别是单一数据集，推荐字段，以及论坛模板。三个类型分别占用三个卡槽做回测，留一个卡槽自己用。然而，回测一段时间后想要从跑完的结果中找出合适的alpha跑二阶时就会发现不同的类型生成二阶的方式都不一样。

**如何解决：**

将一阶的表达式分为计算部分和alpha部分，在生成二/三阶前将alpha部分赋值给一个变量。以下是代码的实现。

```
class Tier2MultiLineAlphaBuilder:    """This class is used to build the second tier multi line alpha expressions.    """    def __init__(self, fields, ops_set, region):        """Constructor of Tier2MultiLineAlphaBuilder        Args:            fields (list):     List of fields to operate on            ops_set (list):    List of operations to apply            region (str):      Region code        """        self.fields = fields        self.ops_set = ops_set        self.region = region    def _format_field(self, field):        """Helper function to remove break line symbols in the field        """        return field.replace('\n', '')    def produce(self):        """Produce the tier 2 multi line alpha expressions.        """        alpha_set = []        for field, decay in self.fields:            if field.split(';')[-1] != 'alpha':                if len(field.split(';')[:-1]) == 0:                    field = 'alpha = ' + self._format_field(field.split(';')[-1]) + ';\nalpha'                else:                    field = ';'.join(field.split(';')[:-1]) + ';\nalpha = ' + self._format_field(field.split(';')[-1]) + ';\nalpha'            lines = field.split(';')            calc_exp = ';'.join(lines[:-1]) + ';\n'            final_exp = 'alpha'            for op in self.ops_set:                final_exp_t2_set = GroupFactory.build(op, final_exp, self.region)                for final_exp_t2 in final_exp_t2_set:                    exp_t2 = calc_exp + 'alpha_t2 = ' + final_exp_t2 + ';\n' + 'alpha_t2'                    alpha_set.append((exp_t2, decay))         return alpha_set
```

**可能遇到的问题**

由于我不是线性去跑一二三阶，所以在get_alpha出来的结果可能会包含二阶和三阶的结果，如果在这个结果集上去做下一阶的生成可能会产出无法运行的alpha，原因是同一个变量在Brain平台不能多次被赋值，为了防止一个表达式导致同批次其他表达式回测失败，我会在正是回测前简单过滤一下语法有问题的表达式

```
def check_exp(exp):    """Check if the expression is valid.    Args:        exp (str): The expression to check    Returns:        bool: True if the expression is valid, False otherwise    """    # Check whether the variable is redefined.    left_vars = []    for line in exp.split(';'):        if '=' in line:            left = line.split('=')[0].strip()            if left in left_vars:                return False              left_vars.append(left)    return True
```
