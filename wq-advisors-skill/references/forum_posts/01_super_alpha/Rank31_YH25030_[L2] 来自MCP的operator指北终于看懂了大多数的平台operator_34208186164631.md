## 理解操作符的作用域 (Scope)

每个操作符都有一个“范围”(Scope)属性，定义了其适用场景。理解这三个范围是正确构建和提交Alpha的关键。

- **`REGULAR`** : 用于构建 **Regular Alpha** 。表达式的核心是返回一个代表对每只股票评分的数值向量。
- **`SELECTION`** : 仅用于 **Super Alpha** 的 **选股表达式** 。结果必须是一个布尔值或数值权重，用于筛选股票池。
- **`COMBO`** : 仅用于 **Super Alpha** 的 **组合表达式** 。定义了如何将多个子Alpha的信号组合成最终的交易信号。

## 算术运算符 (Arithmetic)

### add

- **定义** :  `add(x, y, filter = false)`  或  `x + y`
- **描述** : 对所有输入进行相加（至少需要2个输入）。如果  `filter = true` ，则在相加前将所有输入的 NaN 值视为0。
- **范围** : COMBO, REGULAR, SELECTION
- **示例** :  `add(close, open)`

### multiply

- **定义** :  `multiply(x, y, ..., filter=false)`  或  `x * y`
- **描述** : 对所有输入进行相乘（至少需要2个输入）。如果  `filter = true` ，则在相乘前将 NaN 值视为1。
- **范围** : COMBO, REGULAR, SELECTION
- **示例** :  `close * volume`

### subtract

- **定义** :  `subtract(x, y, filter=false)`  或  `x - y`
- **描述** :  `x - y` 。如果  `filter = true` ，则在相减前将所有输入的 NaN 值视为0。
- **范围** : COMBO, REGULAR, SELECTION
- **示例** :  `high - low`

### divide

- **定义** :  `divide(x, y)`  或  `x / y`
- **描述** :  `x / y` 。
- **范围** : COMBO, REGULAR, SELECTION
- **示例** :  `close / open`

### abs

- **定义** :  `abs(x)`
- **描述** : 返回x的绝对值。
- **范围** : COMBO, REGULAR, SELECTION
- **示例** :  `abs(returns)`

### sign

- **定义** :  `sign(x)`
- **描述** : 返回x的符号。
- **范围** : COMBO, REGULAR, SELECTION
- **示例** :  `sign(returns)`
- **计算方式与解读** :  `sign(x)`  函数返回输入值  `x`  的符号。具体来说：
  - 如果  `x`  是正数，返回  `1` 。
  - 如果  `x`  是负数，返回  `-1` 。
  - 如果  `x`  是零，返回  `0` 。
  - 如果  `x`  是  `NaN` （非数字），则返回  `NaN` 。

### log

- **定义** :  `log(x)`
- **描述** : 返回x的自然对数。用于处理具有极大值或指数增长趋势的数据，使其分布更平缓。
- **范围** : COMBO, REGULAR, SELECTION
- **示例** :  `log(close)`

### power

- **定义** :  `power(x, y)`
- **描述** :  `x` 的 `y` 次幂。可用于增强或减弱信号的强度。
- **范围** : COMBO, REGULAR, SELECTION
- **示例** :  `power(returns, 2)`
- **策略应用** :  `power(x, 0.5)`  (即  `sqrt(x)` ) 可以减弱信号的极端值，而  `power(x, 2)`  可以放大信号的差异。

### signed_power

- **定义** :  `signed_power(x, y)`
- **描述** :  `x` 的 `y` 次幂，但结果的正负号与 `x` 保持一致。计算  `sign(x) * (abs(x) ^ y)` 。
- **范围** : COMBO, REGULAR, SELECTION
- **示例** :  `signed_power(returns, 2)`
- **策略应用** : 当你想放大信号的强度，但又必须保留其原始方向（做多或做空）时，这个算子非常有用。例如， `power(returns, 2)`  会把所有负收益变成正的，而  `signed_power`  则不会。

### sqrt

- **定义** :  `sqrt(x)`
- **描述** : 返回x的平方根。
- **范围** : COMBO, REGULAR, SELECTION
- **示例** :  `sqrt(volume)`

### inverse

- **定义** :  `inverse(x)`
- **描述** : 返回  `1 / x` 。
- **范围** : COMBO, REGULAR, SELECTION
- **示例** :  `inverse(close)`

### reverse

- **定义** :  `reverse(x)`
- **描述** : 返回  `-x` 。
- **范围** : COMBO, REGULAR, SELECTION
- **示例** :  `reverse(returns)`

### max

- **定义** :  `max(x, y, ...)`
- **描述** : 返回所有输入中的最大值（至少需要2个输入）。
- **范围** : COMBO, REGULAR, SELECTION
- **示例** :  `max(high, ts_mean(close, 5))`

### min

- **定义** :  `min(x, y, ...)`
- **描述** : 返回所有输入中的最小值（至少需要2个输入）。
- **范围** : COMBO, REGULAR, SELECTION
- **示例** :  `min(low, ts_mean(close, 5))`

### pasteurize

- **定义** :  `pasteurize(x)`
- **描述** : 如果x是无穷大(INF)或对应的股票不在Alpha的股票池中，则将其设为NaN。
- **范围** : COMBO, REGULAR
- **策略应用** : Alpha表达式经过复杂运算后，可能会产生无效的无穷大值。在最后提交前使用  `pasteurize(my_alpha)`  可以“消毒”，清除这些异常值，提高稳定性。
- **示例** :  `pasteurize(my_alpha)`

### densify

- **定义** :  `densify(x)`
- **描述** : 将一个稀疏的分组字段（如行业代码）变“稠密”，以提高后续分组运算的效率。
- **范围** : COMBO, REGULAR
- **示例** :  `group_rank(returns, densify(subindustry))`
- **策略应用** : 当分组依据的种类非常多（如GICS行业代码有上百个），但在某一天实际出现的种类很少时， `densify` 可以大大加快计算速度。它会临时将当天出现的行业重新编号为 `{0, 1, 2, 3...}` ，避免为那些没出现的行业预留计算资源。

## 逻辑运算符 (Logical)

### and

- **定义** :  `and(input1, input2)`
- **描述** : 逻辑“与”操作。如果两个输入都为真，则返回真，否则返回假。
- **范围** : COMBO, REGULAR, SELECTION
- **示例** :  `trade_when(and(close > vwap, volume > 1e6), 1, 0)`

### or

- **定义** :  `or(input1, input2)`
- **描述** : 逻辑“或”操作。如果任一输入为真，则返回真，否则返回假。
- **范围** : COMBO, REGULAR, SELECTION
- **示例** :  `trade_when(or(returns > 0.05, returns < -0.05), 1, 0)`

### not

- **定义** :  `not(x)`
- **描述** : 逻辑“非”操作。如果x为真，则返回假；反之则返回真。
- **范围** : COMBO, REGULAR, SELECTION
- **示例** :  `trade_when(not(is_nan(close)), 1, 0)`

### if_else

- **定义** :  `if_else(condition, x, y)`
- **描述** : 核心的条件判断工具。如果  `condition`  为真，则返回  `x` ，否则返回  `y` 。
- **范围** : COMBO, REGULAR, SELECTION
- **示例** :  `if_else(close > vwap, 1, -1)`
- **策略应用** : 可以用它实现非常灵活的逻辑。例如，构建一个“增强型”动量因子： `if_else(volume > ts_mean(volume, 20), rank(returns) * 1.5, rank(returns))` 。这个表达式表示：如果今天放量，则动量信号增强50%；否则，保持原样。

### is_nan

- **定义** :  `is_nan(input)`
- **描述** : 如果输入为NaN（非数字），返回真，否则返回假。
- **范围** : COMBO, REGULAR, SELECTION
- **策略应用** : 用于识别并处理数据中的缺失值。例如，用行业均值填充缺失的ROE： `if_else(is_nan(roe), group_mean(roe, industry), roe)` 。
- **示例** :  `if_else(is_nan(close), 0, close)`

### equal

- **定义** :  `input1 == input2`
- **描述** : 如果两个输入相等，返回真，否则返回假。
- **范围** : COMBO, REGULAR, SELECTION
- **示例** :  `trade_when(country == 840, 1, 0)`

### not_equal

## 时间序列运算符 (Time Series)

**核心概念** ：时间序列 ( `ts_` ) 算子是在**“纵向” **上对** 单只股票自身的历史数据**进行计算。可以想象，对于股票A，我们拉出它过去  `d`  天的数据形成一个序列，然后在这个序列上进行计算。

### ts_corr

- **定义** :  `ts_corr(x, y, d)`
- **描述** : 计算 **单只股票** 的两个不同指标  `x`  和  `y`  在过去  `d`  天的 **滚动相关系数** 。
- **范围** : COMBO, REGULAR
- **策略应用** : 这是挖掘 **时序关系** 的核心工具。例如，研究价量关系  `ts_corr(returns, volume, 20)` ，可以检验“放量上涨”或“价量背离”等经典逻辑。
- **计算方式与解读** :
  1. **什么是“滚动”?**   `ts_`  前缀代表“时间序列”。所有  `ts_`  算子都是在一个长度为  `d`  的“滚动窗口”上计算的。想象一个长度为d天的尺子，每天向前移动一格来框选最近d天的数据进行计算。
  2. **什么是“相关系数”?**  它衡量两组数据在一段时间内走势的相似程度，结果被标准化到  **-1 到 1**  的区间内。
  - **接近 1** :  `x`  和  `y`  具有很强的正相关关系（同涨同跌）。
  - **接近 -1** :  `x`  和  `y`  具有很强的负相关关系（一个涨另一个跌）。
  - **接近 0** :  `x`  和  `y`  之间几乎没有线性关系。
  3. **内部计算** :  `ts_corr`  是标准化的协方差，其等价展开为  `ts_covariance(x, y, d) / (ts_std_dev(x, d) * ts_std_dev(y, d))` 。通过除以各自的波动（标准差），消除了量纲影响，只关注形态。

### ts_std_dev

- **定义** :  `ts_std_dev(x, d)`
- **描述** : 计算一个指标在过去 `d` 天自身的 **历史波动性** 。
- **范围** : COMBO, REGULAR
- **策略应用** : 衡量波动率最核心的工具。一个经典的 **低波动率因子** 可以写成  `rank(-ts_std_dev(returns, 20))` ，它会买入过去20天回报率最稳定的股票。
- **计算方式与解读** : “标准差”是衡量数据点离散程度的常用指标。 `ts_std_dev(x, d)`  的计算步骤如下：
  1. **计算滚动均值** : 首先，计算  `x`  在过去  `d`  天的简单平均值  `ts_mean(x, d)` 。
  2. **计算每日离差** : 对于这  `d`  天中的每一天，计算当天值与这个均值的差  `(x_i - 均值)` 。
  3. **计算离差平方** : 将每天的离差进行平方  `(x_i - 均值)²` 。
  4. **求和并平均（方差）** : 将这  `d`  个离差平方加起来，然后除以  `d` （或 `d-1` ），得到“方差”。
  5. **开平方（标准差）** : 最后，取方差的平方根，得到标准差。  **一句话** ：标准差越大，说明  `x`  在过去  `d`  天的波动越大，走势越不稳定。

### ts_zscore

- **定义** :  `ts_zscore(x, d)`
- **描述** : 计算时间序列Z-score，衡量一个指标当前值偏离其 `d` 天历史均值的程度。
- **范围** : COMBO, REGULAR
- **策略应用** : 用于识别 **历史性的极端偏离** ，是构建 **均值回归** 策略的利器。它回答了“当前值偏离历史正常的程度，是其历史波动的几倍？”
- **计算方式与解读** : 在理解了  `ts_mean`  和  `ts_std_dev`  之后， `ts_zscore`  就非常直观了。
  - **公式** :  `(当前值 - d日滚动均值) / d日滚动标准差`
  - **解读** : 结果的含义是“当前值偏离了几个标准差”。Z-score的绝对值越大，说明当前值越偏离其历史正常波动的范围，是一个越“极端”的事件。

### ts_product

- **定义** :  `ts_product(x, d)`
- **描述** : 计算  `x`  在过去  `d`  天的 **滚动连乘积** 。
- **范围** : COMBO, REGULAR
- **策略应用** : 主要用于计算 **复利回报** 。
- **计算方式与解读** : “滚动连乘积”是指在  `d`  天的滚动窗口内，将所有数值相乘。例如， `ts_product(x, 3)`  在今天  `t`  的计算结果是  `x(t) * x(t-1) * x(t-2)` 。 这个功能在金融中至关重要，因为计算一段时间的 **复利回报率** ，就需要将  `(1 + 日回报率)`  进行连乘。例如，要计算过去20天的总回报，不能用  `ts_sum(returns, 20)` （这是单利），而必须用  `ts_product(1 + returns, 20) - 1` 。

### ts_decay_linear

- **定义** :  `ts_decay_linear(x, d)`
- **描述** : 计算  `x`  在过去  `d`  天的 **线性衰减加权移动平均** 。
- **范围** : COMBO, REGULAR
- **策略应用** : 相比于  `ts_mean` （简单平均）， `ts_decay_linear`  给予近期数据更高的重要性，认为**“越近的越重要”**，能更快地响应近期变化。
- **计算方式与解读** : 它给最近的值（今天）赋予权重  `d` ，昨天的值权重为  `d-1` ，以此类推，直到  `d`  天前的值权重为  `1` 。然后将这些加权后的值相加，再除以权重的总和。
  - **公式** :  `(d*x_t + (d-1)*x_{t-1} + ... + 1*x_{t-d+1}) / (d*(d+1)/2)`
  - **示例** :  `ts_decay_linear(x, 3)`  的计算是  `(3*x_t + 2*x_{t-1} + 1*x_{t-2}) / 6` 。

### ts_covariance

- **定义** :  `ts_covariance(y, x, d)`
- **描述** : 返回y和x在过去d天的 **滚动协方差** 。
- **范围** : COMBO, REGULAR
- **计算方式与解读** : “协方差”衡量两个变量在一段时间内是否会“同步变化”。
  - **正协方差** : 两个变量倾向于在同一时间高于或低于它们各自的平均水平（例如，收益率上涨时，成交量也倾向于放大）。
  - **负协方差** : 一个变量高于其均值时，另一个变量倾向于低于其均值（例如，收益率上涨时，波动率倾向于下降）。 “滚动” ( `ts_` ) 协方差意味着这个计算是在一个  `d`  天的移动窗口上连续进行的，反映了两个变量短期内的动态关系。

### ts_kurtosis

- **定义** :  `ts_kurtosis(x, d)`
- **描述** : 返回x在过去d天的 **滚动峰度** 。
- **范围** : COMBO, REGULAR
- **计算方式与解读** : “峰度”衡量数据分布“尾部”的厚度，即 **极端值出现的频率** 。
  - **高峰度** : 意味着分布的尾部比正态分布更“厚”，出现极端值（暴涨或暴跌）的概率更高。金融资产的回报率通常都具有高峰度的特征。
  - **低峰度** : 尾部更“薄”，极端值很少出现。 “滚动” ( `ts_` ) 峰度是在一个  `d`  天的移动窗口上计算该值，可以观察资产回报率分布的尾部风险是否在近期发生了变化。

## *以下为其他时间序列算子，原理与上述类似，不在此赘述其“滚动”特性。*

### ts_max_diff

- **定义** :  `ts_max_diff(x, d)`
- **描述** : 返回  `x - ts_max(x, d)` ，即当前值与d天内最大值的差。
- **范围** : COMBO, REGULAR

### ts_scale

- **定义** :  `ts_scale(x, d, constant = 0)`
- **描述** : 将时间序列数据缩放到[0,1]区间，即  `(x - ts_min(x, d)) / (ts_max(x, d) - ts_min(x, d)) + constant` 。
- **范围** : COMBO, REGULAR

### ts_quantile

- **定义** :  `ts_quantile(x, d, driver="gaussian")`
- **描述** : 计算时间序列分位数，先计算ts_rank，然后将排名应用到指定分布的反累积密度函数上。
- **范围** : COMBO, REGULAR

### ts_arg_max

- **范围** : COMBO, REGULAR

### ts_arg_min

### ts_target_tvr_decay

## 截面运算符 (Cross Sectional)

**核心概念** ：截面 (Cross Sectional,  `cs_`  或无前缀) 算子是在**“横向” **上对** 同一天的所有股票**进行计算。可以想象，在今天这个时间点，我们把所有股票的某个指标值（如  `returns` ）放在一起，形成一个集合，然后在这个集合上进行计算（如排序、标准化）。

### rank

- **定义** :  `rank(x, rate=2)`
- **描述** : 对输入进行截面排序，返回一个在[0.0, 1.0]之间均匀分布的值。
- **范围** : COMBO, REGULAR
- **策略应用** : 将绝对数值转换为相对强弱的排名，是消除异常值、降低回撤最有效的手段之一。
- **计算方式与解读** :  `rank`  将一列无序的数字（比如今天所有股票的收益率）转换成它们的 **百分位排名** 。
  - **步骤1** : 将所有股票的  `x`  值从低到高排序。
  - **步骤2** : 为每个股票分配一个0到1之间的分数。最低的股票得分接近0，中位数的股票得分0.5，最高的股票得分接近1。
  - **示例** : 假设有5只股票，它们的收益率分别是  `[-0.02, 0.05, 0.01, -0.03, 0.02]` 。
  - 排序后:  `-0.03, -0.02, 0.01, 0.02, 0.05`
  - `rank` 后的值 (近似):  `0.1, 0.3, 0.5, 0.7, 0.9`  (这是一个简化的例子，实际分布更均匀)。
  - **作用** : 它忽略了绝对值的大小（-0.03和-0.02之间的差距，远小于0.02和0.05的差距），只关心它们的相对顺序。这使得因子对极端值（比如某只股票涨了50%）不那么敏感，增加了Alpha的稳定性。

### zscore

- **定义** :  `zscore(x)`
- **描述** : 计算截面Z-score， `(x - mean(x)) / std(x)` 。
- **范围** : COMBO, REGULAR
- **策略应用** : 将所有股票在 **同一天** 的某个指标值，转换为以全体股票的均值和标准差为基准的标准化分数。它回答了“这只股票今天的这个指标，相比于所有其他股票，有多极端？”
- **计算方式与解读** :
  1. **计算截面均值** : 计算当天所有股票  `x`  值的平均值。
  2. **计算截面标准差** : 计算当天所有股票  `x`  值的标准差。
  3. **Z-Score** :  `(个股x值 - 截面均值) / 截面标准差` 。

### winsorize

- **定义** :  `winsorize(x, std=4)`
- **描述** : 对截面数据进行缩尾处理，将超出  `std`  倍标准差的极端值拉回边界值。
- **范围** : COMBO, REGULAR
- **策略应用** : 在计算  `zscore`  或进行中性化之前，使用  `winsorize`  可以有效 **降低极端异常值对整体统计（均值、标准差）的扭曲** ，增加Alpha的稳定性。

### scale

- **定义** :  `scale(x, scale=1, longscale=1, shortscale=1)`
- **描述** : 对头寸进行缩放以满足特定的资金规模（booksize）。可以为多头和空头设置不同的缩放比例。
- **范围** : COMBO, REGULAR

### normalize

- **定义** :  `normalize(x, useStd = false, limit = 0.0)`
- **描述** : 对截面数据进行中心化，即减去所有股票在当日的均值，使得处理后的数据均值为0。
- **范围** : COMBO, REGULAR

### vector_neut

- **定义** :  `vector_neut(x, y)`
- **描述** : 向量中性化。寻找一个新向量x*，使其与向量y正交，即剥离y对x的影响。
- **范围** : COMBO, REGULAR

## 分组运算符 (Group)

**核心概念** ：分组 ( `group_` ) 算子是 **截面运算的延伸** 。它同样是在**“横向” **上对** 同一天的所有股票 **进行计算，但计算范围被限定在** 一个个更小的“分组”**内（如行业、国家、市值分层）。可以想象，在今天，我们先把所有股票按行业分成不同的小组，然后在每个小组内部独立地进行排序、标准化等操作。

### group_neutralize

- **定义** :  `group_neutralize(x, group)`
- **描述** : 对  `x`  进行分组中性化，即在每个组内，减去该组的均值。
- **范围** : COMBO, REGULAR
- **策略应用** : 这是构建 **行业中性** 等策略最核心的算子。它能帮你找到“矮子里的将军”和“将军里的士兵”，剔除行业本身带来的影响。
- **计算方式与解读** : 它通过在每个小组内减去该组的均值，来消除组间差异，只关注组内优劣。
  - **步骤1** : 根据  `group`  字段（如  `industry` ）将所有股票分成不同的小组。
  - **步骤2** :  **独立地** 为每个小组计算其  `x`  值的平均数。
  - **步骤3** : 从每个股票的  `x`  值中，减去它 **所属小组** 的平均数。
  - **示例** : 假设我们有4只股票，指标为ROE：
  股票
  行业
  ROE
  中性化计算
  结果
  A
  科技
  15%
  `15% - 13%`
  `+2%`
  B
  科技
  11%
  `11% - 13%`
  `-2%`
  C
  金融
  8%
  `8% - 7.5%`
  `+0.5%`
  D
  金融
  7%
  `7% - 7.5%`
  `-0.5%`
  - *科技组的平均ROE是  `(15% + 11%) / 2 = 13%`*
  - *金融组的平均ROE是  `(8% + 7%) / 2 = 7.5%`*
  - **结果解读** : 尽管股票A的绝对ROE(15%)远高于股票C(8%)，但中性化后，我们发现A和C都是各自行业内的优等生（结果为正）。这样构建的Alpha不会因为整个科技行业景气度高而无脑买入所有科技股，实现了行业中性。

### group_rank

- **定义** :  `group_rank(x, group)`
- **描述** : 在指定的组（group）内对  `x`  进行排序，返回组内排名。
- **范围** : COMBO, REGULAR
- **策略应用** : 寻找组内相对强弱。例如， `group_rank(returns, industry)`  会告诉你一只股票在它所属的行业里，当天的收益排在什么位置。
- **计算方式与解读** : 它在每个小组内部进行独立的截面排名。
  - **步骤1** : 根据  `group`  字段将股票分组。
  - **步骤2** : 在每个小组内部，对股票的  `x`  值进行从低到高的排序，并分配一个0到1的百分位排名。
  - **示例** : 使用上面  `group_neutralize`  的例子：
  股票
  行业
  ROE
  `group_rank`  结果
  A
  科技
  15%
  ~0.9 (科技组内最高)
  B
  科技
  11%
  ~0.1 (科技组内最低)
  C
  金融
  8%
  ~0.9 (金融组内最高)
  D
  金融
  7%
  ~0.1 (金融组内最低)
  - **结果解读** :  `group_rank`  同样突出了A和C是各自组内的佼佼者，但它将结果直接归一化到0-1之间，使得不同组的结果可以直接横向比较。

### group_zscore

- **定义** :  `group_zscore(x, group)`
- **描述** : 在组内计算Z-score。
- **范围** : COMBO, REGULAR
- **策略应用** : 与 `zscore` 类似，但它是在一个更小的范围内（如行业内）进行标准化。它回答了“这只股票今天的指标，相比于它同行业的其他股票，有多极端？”
- **计算方式与解读** :
  1. **分组** : 将股票按  `group`  字段（如  `industry` ）分成不同的小组。
  2. **组内计算** : 在每个小组内部，独立地计算该组的均值和标准差。
  3. **组内Z-Score** :  `(个股x值 - 组内均值) / 组内标准差` 。

### group_std_dev

- **定义** :  `group_std_dev(x, group)`
- **描述** : 计算  `x`  在组内的标准差。
- **范围** : COMBO, REGULAR
- **计算方式与解读** : 在每个分组内部，独立计算指标  `x`  的截面标准差。可用于识别不同行业/板块的波动率差异。

### group_cartesian_product

- **定义** :  `group_cartesian_product(g1, g2)`
- **描述** : 创建一个交叉分组，例如，将国家和行业合并成“国家-行业”这个新分组。
- **范围** : COMBO, REGULAR
- **策略应用** : 用于进行 **双重甚至多重中性化** 。例如，在欧洲市场，同时对国家和行业进行中性化是常见的做法:  `group_neutralize(returns, group_cartesian_product(country, industry))` 。

### group_scale

- **定义** :  `group_scale(x, group)`
- **描述** : 在组内进行缩放，将  `x`  的值归一化到 [0, 1] 区间。
- **范围** : COMBO, REGULAR

### group_mean

- **定义** :  `group_mean(x, weight, group)`
- **描述** : 计算  `x`  在组内的（加权）平均值。
- **范围** : COMBO, REGULAR

### group_sum

- **定义** :  `group_sum(x, group)`
- **描述** : 计算  `x`  在组内的总和。
- **范围** : COMBO, REGULAR

### group_count

- **定义** :  `group_count(x, group)`
- **描述** : 计算组内  `x`  的有效值（非NaN）的数量。
- **范围** : COMBO, REGULAR

### group_backfill

- **定义** :  `group_backfill(x, group, d)`
- **描述** : 在组内进行回填，用最近  `d`  天的组内均值填充NaN值。
- **范围** : COMBO, REGULAR

## 转换与其他运算符

### bucket

- **定义** :  `bucket(x, buckets)`  或  `bucket(x, range)`
- **描述** : 将连续的浮点数值按指定方式分割成离散的“桶”，生成一个可用于分组的字段。
- **范围** : COMBO, REGULAR
- **策略应用与解读** : 这是 **使用连续数据（如市值、估值）作为分组依据的唯一正确方法** 。例如， `bucket(cap, 10)` 会创建10个市值从低到高的分组（桶），然后你可以用它作为 `group_rank` 的第二个参数，计算每个市值层级内部的因子排名。
- **示例** :  `group_rank(returns, bucket(cap, 10))`
- **详细说明** :  `bucket` 用于将浮点数转换为离散的桶索引，以便用于分组操作。可以通过 `buckets` 参数指定具体的分割点，或通过 `range` 参数指定一个范围和步长。默认情况下，会自动添加两个覆盖 `(-inf, start]` 和 `[end, +inf)` 的隐藏桶，可以使用 `skipBegin` 和 `skipEnd` 参数移除它们。

### trade_when

- **定义** :  `trade_when(condition, x, z)`
- **描述** : 当条件  `condition`  为真时，返回  `x` ，否则返回  `z` 。
- **范围** : COMBO, REGULAR
- **策略应用与解读** : 用于 **控制交易逻辑和仓位** 。例如， `trade_when(adv20 > 1e6, my_alpha, 0)` 表示只在成交额大于100万时，才计算 `my_alpha` 信号，否则信号为0（即不交易），可以有效过滤掉流动性差的股票。
- **示例** :  `trade_when(close > vwap, 1, 0)`
- **详细说明** :  `trade_when(交易触发条件, Alpha表达式, 退出条件)` 。如果退出条件为真，则Alpha值为NaN（平仓）；否则，如果交易触发条件为真，则Alpha值为 `Alpha表达式` ；否则，Alpha值保持不变。此算子有助于降低相关性和换手率。

### vec_sum / vec_avg

- **定义** :  `vec_sum(x)`  /  `vec_avg(x)`
- **描述** : 对向量化数据（例如，一个字段包含多个值）进行求和或求平均。主要用于处理特定格式的财务报表数据。
- **范围** : COMBO, REGULAR
- **示例** :  `vec_avg(financial_statement_field)`

### combo_a

- **定义** :  `combo_a(alpha, nlength = 250, mode = 'algo1')`
- **描述** : Super Alpha中的组合函数，通过平衡每个子Alpha的历史回报与最近n天内的波动性，将多个Alpha信号组合成一个加权输出。
- **范围** : COMBO
- **示例** :  `combo_a(alpha1, alpha2)`

### self_corr

- **定义** :  `self_corr(input)`
- **描述** : 计算输入矩阵的自身相关性。输入一个(D x N)矩阵，输出一个(D x N x N)的相关性矩阵。主要用于Super Alpha。
- **范围** : COMBO
- **示例** :  `self_corr(returns)`

### reduce_*

- **定义** :  `reduce_sum(input)` ,  `reduce_avg(input)` ,  `reduce_max(input)` , etc.
- **描述** : 一系列聚合函数，用于对2D或3D矩阵的最后一个维度进行计算。例如， `reduce_sum` 对最后一个维度求和。主要用于Super Alpha。
- **范围** : COMBO
- **示例** :  `reduce_sum(some_matrix)`
