我截止10月初value factor是0.88（计算期间是2024.6~2024.8），当时正好是ATOM比赛前期，还享受到了GLB的theme加成，所以base收入很高，但由于本人在2024.9没有提交alpha，因此在11月份我的value factor骤减到了0.13（计算期间是2024.7~2024.9），从上图可以看出来，在0.13的value factor下不管做什么，收入都非常的微薄，基本上一天1usd。

10~11月份由于是ATOM比赛期间，ATOM积分机制是25%的IS+75%的OS，我按照比赛的机制+我自己的一些方法论，优化每一个alpha与组合的IS与OS，更重要的是，提交了大量且diverse的alpha，因此value factor得到了提升，最近更新后直接升到0.94（这也与我比赛的os表现吻合）。关于这些方法，大家可以参照上方我分享的文章，里面描述了我在比赛之中的整体思路，也欢迎大家探讨。

接下来会分享一些具体的思路：

我参加过IQC,MAPC,ATOM，就我的经验来看，IS加分是有投资组合优化的意义的。在平时的alpha调整之中，如果有个“这样调整在比赛中会加分”的思路，对提升alpha的质量是很有帮助的。

**2.人工制作alpha：分类进行处理。**

人工制作的alpha往往具备 **强经济学意义** 与 **强解释性** 。

关于解释性：

平时大家做alpha往往以跑机器为主，跑出来之后可能会进行手动的调整。这样的方法难以避免的是alpha的可解释性受质疑（有一些alpha基于有经济学意义的模板和与模板原数据字段类似的数据字段组成的搜索空间，这种保留了一定的可解释性）。

不过个人理解下可解释性更多是在实盘遇到回撤的时候，pm需要考虑的问题，并不代表这个alpha的OS表现就一定不如强解释性的alpha。

关于经济学意义：

如果一个alpha的收益来源是基于research based的市场背离，比如说风险补偿与错误定价，并且经过长时间的检验（回测），且考虑了不同的风格（中性化）都具有统计学意义的显著，那么这个alpha在未来一段时间内快速衰减的概率是比较小的。这也是为什么需要人工制作有经济学意义的alpha。

在ATOM比赛期间我130个alpha之中，有10%左右的alpha都是人工制作的。这里也分了两种，第一种是基于一个数据字段出发，根据自己的经验猜测其数据分布，然后fitting出来的alpha，第二种是根据论文、研报、以及中文论坛的alpha灵感复现出来的alpha。

对于第一种，当然是需要进行严格的检验，检验的方法可以参考上文链接。

对于第二种，基本上就不会再用那些方法去检验，更多的是看跟组合的self correlation以及IS的加分。

**3. 机器+人工制作的alpha：基于灵感进行制作**

这种方法不同于纯人工制作，也不是机器跑出来之后手动调整，可以理解为使用机器筛选出有预测信号的数据字段及其适用的表达式后，基于一定的经济学意义对其进行更进一步的演化。

举个例子：

```
s_log_1p(group_zscore(group_rank(ts_rank(ts_backfill(subtract(vec_sum(anl16_actvalue_normal),vec_sum(anl16_estimate_d1_estvalue_normal)),63),500),densify(subindustry)),densify(sector)))
```

这个alpha在我比赛的中期，也就是平均一个alpha只能加300~500的时候，给我加了900分。

这个alpha最初通过模板跑出来，其实是这样的（这个alpha已经可以提交，不过表现不够好）：

```
s_log_1p(group_zscore(group_rank(ts_rank(ts_backfill(vec_sum(anl16_actvalue_normal),63),500),densify(subindustry)),densify(sector)))
```

其中 anl16_actvalue_normal 的含义是

*Normalized actual value in the fiscal report.*

首先它是经过标准化处理的，其次它表示的是财报上的实际值。这时候就很容易想到，analyst surprise类型的alpha，往往就是实际值与预期值的差值（这里面会分盈利预期、股票价格预期等等），由此在这个数据集里面找到了:

*anl16_estimate_d1_estvalue_normal*

从normal以及d1这些部分可以知道，这个是处理过后的，滞后一期的分析师预测数据，并且也经过了标准化处理。

因此就直接使用subtract处理（这里个人尝试过regression_neut, ts_regression, divide, 都能做出可以提交的alpha，且相比于纯粹的actual value有信息增益)

类似的思路可以延申到：nlp分析情绪的good 与 bad。

那么对于这类思路制作的alpha，其主信号是具备经济学意义的，外层的表达式是用来提炼主信号的增强，相比于纯粹机器跑出来的alpha，或者纯粹手动fitting的情况下，我会更放心的提交（加分多的情况下）

### 评论 #8 (作者: TL87739, 时间: 1 year ago)

[XL31477](/hc/en-us/profiles/8979998652567-XL31477)

感谢您的回复。

关于您提到的几个问题：

1. 调整中性化/decay的通用技巧：

**关于decay：** 首先要看这个alpha使用数据的类型，对于更新频率较低的数据（包括但不限于基本面、分析师、事件数据），这些数据基本上是利用股票的长期特征进行预测，那么可想而知是可以以一个更大长度的步长去调整decay的，尤其是一些基本面类型的alpha，它本身的预测信号就应该是覆盖较长未来时间，如果我拉大decay（相当于混入更长滞后期的信号）对他的performance造成了很大影响，那反而要去质疑他的稳健性了。

对于更新频率较高（量价为主）的数据，一般再decay上都是5日为步长，甚至是用hump\ts_weighted_decay这样的operators进行微调。

总的来说decay的调整以获得更高的is分数为目标。

**关于中性化：** 可以使用visualization功能，看alpha的收益在不同市值、行业、地区下的分布，以此判断。对于ATOM的单一数据集限制，无法自己做risk-factor。但是在平时研究之中，是可以做一些通用的risk-factor，比如20日收益率（反转因子），Ret(252,21)(动量因子），来进行中性化。

2.如何判断alpha是否适合增加decay：我认为上面的回答也回答了这个问题

3.如何预估增加decay后带来的drawdown风险：

关于这个问题，首先有一部分的alpha在增加decay后表现不仅不会下降，反而会上升（这一点在基本面类型的alpha较为常见）

对于表现变差的alpha，在比赛之中，会试图取得加分与表现的中位数。在平时研究之中，会试图取得margin与drawdown的平衡（以margin高于15为目标）

4.关于比赛环境中人工alpha的匹配：

我在回复 [LO56106](/hc/en-us/profiles/8459306799511-LO56106) 的回答之中也谈到了我的想法，就是人工和机器是可以同步进行，跑代码的同时也可以进行alpha尝试。另外我比赛之外的时间也会有个人的alpha研究，有灵感就会在brain上复现，总的来说我认为以机器为主导，人工为辅助，是可以适应比赛节奏的。

5. 在寻找灵感和演化alpha的技巧资源：

首先阅读论文与研报是最快的寻找灵感的方法。至于以什么方向去读，有一种方法是以brain的数据出发，先找感兴趣的数据，然后google搜相关论文进行研究。

也可以先跑一遍信号，然后对信号进行研究。

另外读研报多的话，一般可以发现alpha也是有类别的，可以顺着一个类别不断的读下去，会有更多的灵感和技巧。
