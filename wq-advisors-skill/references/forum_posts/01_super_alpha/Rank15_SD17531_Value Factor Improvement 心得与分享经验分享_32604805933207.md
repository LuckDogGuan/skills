## 帖子正文

前车之鉴,后车之覆.

本人是从2024.12.14开始提交alpha的,VF分别是0.5- 0.23- 0.42- 0.46.
看alpha提交数量:
12月份提交了35个alpha,其中JPN 2个,ASI 15个, USA 18个.此月因为没有40个alpha,也没有满足20天提交,所以没有更新.
25年开始,1月份提交了36个alpha,其中TWN 3个,HKG 3个, ASI 17个,USA 13个.这次满足20天提交, 但是数量不够40个. 加上12月份的alpha,数量上足够了.更新第一次VF 0.23,combined0.04.总结:USA,ASI勉强足够数量了,但是TWN,HKG,JPN这三个地区数量不够,不够robust.且第一次更新OS的时候,发现这三个小地区的OS几乎都是负数!而且三个小地区在计算combined指标的时候,是等权的,也因为后续没有这三个小地区的权限,所以这三个小地区会持续影响我的各种指标,现在好想回到过去给自己一巴掌.

2月份

提交了36个alpha, EUR 15个, GLB 4个,ASI 9个, USA  8个.这个月的数量依旧没有40个.因为小地区的权限没有了,所以开始提交其他大地区.其中EUR勉强数量过得去,但是GLB数量少.USA,ASI继续提交,增加了robust.本月更新VF 0.42,combined0.72, combined select 1.33.可以看到combined提升挺大,selected从无到有.Genius评级到这个月份的数据,其实满足了Master的要求,但是更新的时候已经很晚了,时间不够去把alpha数量刷到120个,所以无法满足Master要求,最终定级Expert.

3月份

提交了33个alpha,其中EUR 18个,GLB 3个,ASI 6个,USA 6个.这个月数量不足40个, EUR总数量33个,比较稳定了,不管是好的很稳还是差的很稳,反正很稳;GLB数量依旧不够,全部只有7个.ASI,USA在原先基础上增加.可以看到,这个月的alpha有一个EUR是肯定增加了robust的, 其他几个地区继续修补.数据更新, VF 0.46, combined 0.98, combined selected 1.1.可以看到combined是继续提升的,selected因为不确定数量,所以增加降低,可能是因为实盘的影响.

4月份待更新

前车之鉴,后车之师.

其实从上面的各种数据来看, 小伙伴们应该可以看出不少问题来了.
quantity is quality.可以明显看到我每个月提交的alpha数量是很稳定35个左右, 而Master的要求是3个月提交120个,平均每个月40个.而且,还有群友提出,第一次更新VF的时候, 要求已提交的alpha数量是40个还是50个. 建议大家每个月至少提交40个alpha吧,不然发现自己满足Master的performance,却不满足signal数量要求, 那很尴尬,因为上次定级明显是只有数量不满足,其他指标都满足Master的,起码损失一万块.

其次,勤奋很重要.我这三个月提交的alpha,其实都是因为懒惰,关于代码工程,其实很早就已经写好了大部分的功能需求,但是一直没有去动手写;另外就是模板问题,其实我做了很多模板,也出了不少alpha,但是一直没有去优化,其实很多都可以持续完善的.在原有的基础上推陈出新.

最后,Robust检测很重要,换手率控制很重要,这部分是我参考群友帖子写的代码,确实有很好的收获.以前提交alpha总是乱来,现在科学了很多.在这个过程中,还是学到了很多,也把很多原先过不了prod corr的alpha完善到通过提交标准.

1.遍历neut

```
for alpha in new_query_alpha:    print(f"当前处理的 alpha: {alpha}")    if alpha['region'] == 'USA':        neut_list = ['STATISTICAL', 'CROWDING', 'FAST', 'SLOW', 'MARKET', 'SECTOR', 'INDUSTRY', 'SUBINDUSTRY', 'SLOW_AND_FAST', 'REVERSION_AND_MOMENTUM']    elif alpha['region'] == 'GLB':        neut_list = ['STATISTICAL', 'CROWDING', 'FAST', 'SLOW', 'MARKET', 'SECTOR', 'INDUSTRY', 'SUBINDUSTRY', 'COUNTRY', 'SLOW_AND_FAST', 'REVERSION_AND_MOMENTUM']    elif alpha['region'] == 'EUR':        neut_list = ['STATISTICAL', 'CROWDING', 'FAST', 'SLOW', 'MARKET', 'SECTOR', 'INDUSTRY', 'SUBINDUSTRY', 'COUNTRY', 'SLOW_AND_FAST', 'REVERSION_AND_MOMENTUM']    elif alpha['region'] == 'ASI':        neut_list = ['CROWDING', 'FAST', 'SLOW', 'MARKET', 'SECTOR', 'INDUSTRY', 'SUBINDUSTRY', 'COUNTRY', 'SLOW_AND_FAST', 'REVERSION_AND_MOMENTUM']        # neut_list = ['COUNTRY']    elif alpha['region'] == 'CHN':        neut_list = ['CROWDING', 'FAST', 'SLOW', 'MARKET', 'SECTOR', 'INDUSTRY', 'SUBINDUSTRY', 'SLOW_AND_FAST', 'REVERSION_AND_MOMENTUM']    else:        print(f"未知的 region: {alpha['region']}")        continue
```

2.遍历其他,有些效果一般我就没有继续使用了,比如truncation,设置0.001到0.1都行,decay也自己修改一下.然后如果是用的group,可以修改为其他的group操作符.还有就是如果表达式里面用的是industry这种,也可以遍历替换成subindustry之类的.universe也可以考虑遍历.

```
    truncation_list = [alpha['truncation'],]    # truncation_list = [0.01, 0.1]    decay_list = [0, alpha['decay'],2,6,10]    # decay_list = [0, 5, 10, 15, 20, 25, 30, 35, 40, 45, 50]    group_ops = ['group_zscore', 'group_rank', 'group_neutralize']    # group_ops = ['group_rank',]
```

还有不少可以优化的地方,一般这么处理以后,一定会发现比你原先alpha表现更好的alpha.就是可能需要分成几步去做,不然量太大了.可以考虑自己写一个自动的,找到回测过的最好的neut,然后找到最好的truncation,就这么分层去回测,会大大减少回测量.每次你可以保留你认为的最好的两个结果,然后到下一步继续优化.就是要做自动化,我目前还没有做这一块.

根据全球会议的介绍,现在value factor是4-6周更新一次.一般来说第一次更新是比较久的,比如你4月开始回测,那么大概率在6月份才会更新你4月份的VF.一般是滞后于当前月份2-3个月.然后如果你4月份交的数量比较少,那可能需要到7月份才更新了.第一次更新以后,你可以关注学习群,哪天999+消息,基本上就是有更新了.经验来说,没有固定时间,只能等,这也是为人所诟病的一点.
