## 帖子正文

很多同学对于点亮小数据集都感觉到很头疼，这里分享两个点亮EARNING, RISK金字塔的小技巧，希望对你有所帮助。

**前提条件** ：找到有信号的ATOM因子，在此基础上加上EARNING/RISK数据集的字段。

### **点亮EARNING金字塔**

**模板：**

```
x = x; #x是有信号的ATOM因子open_e = 1;exit_e = ern3_next_interval < <days/> #可自行尝试例如3，5，10等trade_when(open_e,x,exit_e)
```

这里X代表有信号的ATOM因子，ern3_next_interval代表的是trading days before next report date，也就是说在财报公布前若干天把持仓清空。我的理解是财报公布当天股票的波动性大（要么大涨，要么大跌），在财报公布前若干天提前平仓有助于提高策略的稳定性。

**效果：** 和X（ATOM因子）的相关性会较高（0.9+），这个模板适用于想点亮EARNING,或者原来的ATOM因子PC在临界点（如0.71）想进一步降低PC时使用。

### **点亮RISK金字塔**

**模板：**

```
x = x; #x是有信号的ATOM因子vector_neut(x, <risk70/>)
```

X代表有信号的ATOM因子,<risk70/>代表的是risk70(multi-factor model)数据集, 在多个区域中如USA,EUR,GLB都有该数据集。vector_neut对于X进行正交化（中性化处理），和statistical/crowding/fast等类似，都是对于某个ATOM进一步进行风险中性化处理。

**效果：** 和X（ATOM因子）的相关性会较高（0.9+），这个模板适用于想点亮RISK,或者原来的ATOM因子PC在临界点（如0.71）想进一步降低PC时使用。

本来想贴图展示一下自己用这两个模板提交的因子，但是贴图时系统提示错误。大家用自己的ATOM因子跑一下就可以。

希望这两个小技巧可以帮助你点亮更多的塔！ 愿国区小伙伴多多点塔多多冲击GM!~
