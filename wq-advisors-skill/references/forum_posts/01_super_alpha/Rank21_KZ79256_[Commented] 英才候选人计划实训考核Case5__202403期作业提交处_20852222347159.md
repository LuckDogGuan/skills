### 评论 #9 (作者: YH69102, 时间: 2 years ago)

*Task 1 :  **自设一个类“fitness score”***

通过 **阅读< The 101 Ways to Measure Portfolio Performance>考虑将"fitness score"定义为：**

```
(OS_to_IS**0.5) * sharpe * (min(abs(returns)/(drawdown+0.000001),3)/(max(turnover,0.125))*(max(correlation,0.25)))**0.5
```

下面解释一下将公式这样设置的目的：

原本fitness的公式为:

```
sharpe * (abs(returns)/max(turnover,0.125))**0.5
```

首先改进公式第一部分中的sharpe ,为了减少对前10年overfitting的可能，将sharpe进行缩放，缩放幅度为

(OS_to_IS ) ** 0.5

其中OS_to_IS 定义为 :    alpha最后2年的sharpe  /  alpha前8年的sharpe

然后在测试中发现

这个alpha 在最后2年表现不错，在7～8年几乎无效，导致OS_to_IS 的值较大，但实际上这个alpha在2年的时间里几乎无效。说明OS_to_IS还是有局限性，所以在公式中定为 (OS_to_IS)**0.5。

这可以让不同alpha的OS_to_IS 向1靠拢，进而也让不同alpha的OS_to_IS比值更接近1，降低它的影响。

公式的第二部分改进为：

```
(min(abs(returns)/(drawdown+0.000001),3)/(max(turnover,0.125))*(max(correlation,0.25)))**0.5
```

也就是在原fitness的第二部分中引入drawdown,correlation的影响，越低的drawdown、correlation 会有越高的

fitness score

min(abs(returns)/(drawdown+0.000001),3) 的含义是在returns 固定的情况下，对超出一定值的drawdown进行惩罚（阈值为abs(returns) / 3 ,这里的常数可根据对风险的厌恶程度自定义），分母+0.000001 主要是考虑到有些alpha不进行交易所以drawdown=0会报错

(max(turnover,0.125))*(max(correlation,0.25)) 引入了correlation的影响，correlation越低fitness score越高，将correlation与0.25取大值也是为了防止一些极端值的影响 。在外面根号的作用下最小值为sqr(0.25) 最大值为 sqr(1)  这里的比值最多为2

**fitness score 总结：**

优点：改进后的fitness综合了 经OS/IS调整的sharpe  、 returns 、 drawdown 、 correlation 、 turnover.

可以更多的考虑alpha在OS中的表现 并 降低风险。

弊端：通过上面的例子可以看到OS/IS 的调整不够完善，有可能给 那些对最后2年过拟合的alpha 高分

drawdown 的部分需要按需调整具体的值要根据市场，原有alpha pool 调整。

#### 

#### **提交alpha 的其他检验：**

首先可以计算alpha 的 fitness score   并与自己的alpha pool 中的所有同市场的alpha的fitness score 的均值比较，至少需要高于 均值-标准差 。

然后还可以计算before & after 的相关性（这步可能可以不做，有时候alpha是在已有alpha基础上改进的来的self-corr 比较高）

最后可以根据自己alpha pool 最差的表现 进行定向的提升。例如：如果margin过低，则提交使得margin 提升最多的。

**Filtering score**

101performance是从个策略的角度，重点关注收益和风险两个方面，进而度量策略效益，而没有提供指标关注单策略的交易成本以及其和多个策略之间的关系，原本的fitness相比它更多关注了交易成本，而我设定的将仅考虑更多方面

1. **覆盖率分数，** C = (l+s)/total，C应该越大越好，对于相同资金而言，交易数量越少，每只股票的权重越大，那么组合越有可能无法分散风险，因此通过C去惩罚绩效；
2. **相关性分数，** Corr=（1.4-self_corr），相关性越小，绩效得分应该越大，此处为了保证score为正，因此用1.4减去self_corr取正数，而1.4是因为，我最近尝试挖掘对角线的alpha，最低也有0.3-0.4的相关性，取1.4能保证在相关性很小的情况下，作为乘数时，不损失sharpe；
3. **绩效分数，S=sharpe**
4. **换手率，T = max(turnover,0.125)**
5. **回撤分数，D = mean(max_drawdown%- min_drawdown%)，** 这里我计算了drawdown落差而不是直接用max_drawdown，这是因为max_drawdown仅仅度量一个时点，有时容易低估alpha的长期表现，相对来说，使用“落差”，可以反应时序上的变化。这里的计算方式是，对年度drawdown进行两次排序（降序+升序），用降序排列的数组减去升序排列的数组，然后求mean并取绝对值。如果高低落差越小，说明每年的亏损较为平均，可以接受；如果落差较大，说明有一年或有几年出现极端情况，这样说明alpha没有真正相对市场中性。如果有每年回撤都很大的情况，比如每年都10%，这个时候会导致D为0，看上去有bug，会让score变无穷，但是一般这种时候也在sharpe上进行反应了。
6. **分数为(C*S*Corr)/(T*D)，** 看以下我最近交的一个alpha，C=1.1，S=2.22，self_corr=0.31，T=0.2076，D经过测算为2.512，最后score为4.68，效果比较不错

1. 首先是一个共识：当有了分数，就有了fitting，低于分数的，我们不去提交，这会使得我们的Alpha库朝某一方向去进化。fitting不是“坏事”，坏的是我们找了错误的目标进行fitting，portfolio management中有很多单一的目标是很危险的，例如return，可能是个错误的目标，因为并不是return越高越好。从第一性原理出发，我们要尽可能向一个基本“无懈可击”的目标去进行fitting，金融里的圣杯之一就是‘diversity’。因此，我们的分数，有没有让我们的Alpha库更加的diversity,是一个关键的目标，也会是一个比较安全的目标。

2. 多大是大？确实，大的值有无限大。但是我们换个角度去思考，或许会有一些有趣的答案，即小的值可以有极限，例如0.这就像股票投资一样，买一个股票（如果没有杠杆），那么最多可以收入无限大，但是最差只能归零。我们提到，提交Alpha可以有其他额外的检验，将这个思想放入我们分数中，我们可以对好的Alpha进一步分类。即优先提交那些能让我们Alpha pool的correlation降低的Alpha, 对于那些让我们Alpha pool的correlation提升的Alpha，我们可以先留着，以后再考虑怎么提交，或者比赛要冲数量再考虑。Alpha pool(即performance comparison取到的值）是很关键的，假设经过测试，Alpha pool与已提交的Alpha的相关性，最高的时期能到98%。这是多么触目惊心的结果？这说明交了那么多Alpha,还不如就交一个。

3. 计算correlation可能比较耗计算资源，这里替代的也可以考虑把两个Alpha直接做回归取R方，让R方越来越小，它的极值就是0.

4. 没有OS，我们可以自己创造OS。上面一位同学，自留了两年做OS，是很推荐的操作。

### 评论 #26 (作者: HW97336, 时间: 2 years ago)

**Task 1：自设一个类“fitness score”**

核心表达式：

```
sharpe_score = (sharpe_ir * sharpe) ** 0.5reward = 1.1 if after_corr < before_corr else 1corr_score = (1.4 - self_corr) * rewardreturns_score = abs(returns)risk_score = 1 / (drawdown + 0.05)turnover_score = 1 / max(turnover,0.125)score = sharpe_score * corr_score * (returns_score * risk_score * turnover_score)**0.5
```

我将score拆解为五部分——sharpe/corr/returns/risk/turnover：

sharpe部分包括sharpe_ir和sharpe两部分，前者的计算方法是取得目标alpha的pnl，以252天为时间窗口，按照Brain平台上对sharpe的定义滚动计算出sharpe，然后计算时间序列上sharpe的信息比率，即mean(sharpe)/std(sharpe)，加入这部分是为了跟踪sharpe在一年窗口的持续变化，我们期望高sharpe均值和低sharpe波动，再乘以整个时间段的sharpe来衡量综合表现，开根号是因为我们纳入了两个sharpe指标。

corr部分包括self_corr和before_after_corr两部分，前者的计算方法是一对一计算目标alpha的pnl与submitted alpha的pnl的相关系数，以400天为滚动窗口计算均值，然后取所有alpha中最大的那个相关系数，通过1.4-self_corr的方式将其放置在分子位置；后者的计算方法是分别计算before pnl和after pnl与submitted alpha的pnl的相关系数，与之前的方法类似，然后比较二者大小，如果after的相关系数更小，那就给alpha一个奖励倍数reward，这里设置的是1.1，原本打算乘以差值，但一般来说一个alpha能够降低的相关性是比较小的，直接乘以固定倍数的处理更加简单。

drawdown部分计算各年份的drawdown平方和均值，再开根号，相比于直接求均值，还是为了放大高drawdown的影响，通过加一个常数避免drawdown过小导致score过大。
