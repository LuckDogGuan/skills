修复方案： 建议将 subprocess.Popen 的调用方式从 字符串形式 改为 列表 (List) 形式 。这样 Python 会调用 Windows 的 CreateProcess API 自动处理参数转义，无需手动处理复杂的 cmd 引号逻辑。

   ```
2. 修改 open_submitter 函数（约 2284 行）：
   ```
   # 修改前：subprocess.Popen(    f'cmd /k "{APP_PYTHON}" "alpha_submitter.py"',    cwd=simulator_dir,    env={**APP_ENV, **submitter_env},    creationflags=subprocess.CREATE_NEW_CONSOLE)​# 修改后（推荐）：subprocess.Popen(    ["cmd", "/k", APP_PYTHON, "alpha_submitter.py"],    cwd=simulator_dir,    env={**APP_ENV, **submitter_env},    creationflags=subprocess.CREATE_NEW_CONSOLE)
   ```
3. 修改 open_hk_simulator 函数（约 2370 行）：
   ```
   # 修改前：subprocess.Popen(    f'cmd /k "{APP_PYTHON}" "autosimulator.py"',    cwd=hk_simulator_dir,    env=APP_ENV,    creationflags=subprocess.CREATE_NEW_CONSOLE)​# 修改后（推荐）：subprocess.Popen(    ["cmd", "/k", APP_PYTHON, "autosimulator.py"],    cwd=hk_simulator_dir,    env=APP_ENV,    creationflags=subprocess.CREATE_NEW_CONSOLE)
