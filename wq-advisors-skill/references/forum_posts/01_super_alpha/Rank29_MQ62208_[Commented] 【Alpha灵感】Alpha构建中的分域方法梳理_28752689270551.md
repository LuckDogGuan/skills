***1. 什么是"分域"***

“分域”是指按某种方式将市场划分为不同的区域，在这些区域内对datafield或Alpha进行额外的处理。该区域的划分通常是通过公司特征，比如行业（Sector, Subindustry），市值大小，也可以由研究团队自行定义，例如通过机器学习动态聚类。

**注：** 大-小市值的界限可以由研究员自己定义，比如按照截面市值分位数，每N%划分一个界限，小于该界限认为是小市值，在经典的Fama-French 3因子模型中，N=30%

***2. 为什么要分域***

分域的目的主要是为了对datafield或Alpha进行 **差异化的处理** 。

1. 传统多因子模型，是按照股票因子值在全市场对股票本身进行排序、打分——这种方式将所有股票同等看待。通常，不同股票的基本面、量价等属性存在较大区别。 低估值、规模较大、持续支付股息的蓝筹股，和高估值、规模较小、股息支付不稳定的成长股，可比性较差。因此，对股票分域建模显得尤为重要。[1]
2. 例如，我们在进行fundamental数据进行研究时，我们需要考虑到不同行业基本面属性差别较大。银行股股息率较高，如果在不做处理的情况下按股息率指标选股，那么可能选出的都是银行股。如果银行股股价下跌，那么我们的投资组合会随之回撤。对该类问题，常见的解决方法是行业中性化，理念是我在每个行业中都选出股息率较高的若干股票构成投资组合，从而规避行业风险带来的影响。

借鉴研报中的框架，我们可以看到因子分域的基本方式。我们主要讨论因子计算层面的分域，从维度上来看，可以分为截面型和时序型；从处理手段上来看，可以分为离散型和连续型，上文提到的行业中性化是离散型的一种。

**首先我们来关注截面-离散分域，该种方法强调当前时点下个股与个股间的横向差异。以下是一些常见的应用和案例。**

1. **我们上文中提到的行业中性化，在行业内进行打分。** 具体的做法是我们将每一个股票对应到某个特定行业上，在这个行业中进行排名，这个业界常用的做法之一，同时还有行业内减掉均值，行业内zscore以及设置行业哑变量回归取残差。常见的行业分类：CHN的中信一级行业、USA的GICS等
2. **通过0/1变量、哑变量的方式对股票进行筛选。** 普通的量化研究做法可能是在整个市场中构建信号再选股，但这样通常会有一些问题，例如流动性较差导致交易实施难，即使Alpha很出色，但回测效果是失真的；为了解决这种方法，我们可以对股票池进行特定筛选，比如在Brain上，我们可以看到TOP3000，TOP2000等，或者平时我们常听的S&P500，罗素2000，沪深300。这些都是分域的产物，定期对股票的特征进行一些判断从而考虑是否进行交易。这便于研究者做出差异化的策略，更好实现资产配置需求。
3. 最后是对区域中的股票取一些统计量（均值、最大值），再用于其他计算，这其实是行业中性化的衍生。
4. 相关的运算符：group_zscore, group_rank, group_max, group_mean, group_median, group_vector_neut等

而截面-连续分域大同小异，就和微分一样，我们只需要找到一种方法将域无限可分。一种常见的做法是设置激活函数，通过激活函数给另一个指标进行重新加权，从而控制个股在截面上的排名。在[1]中提供了一种模板

Multiply(<indicator1>, <act func>(<indicator2>))

其中indicator1和indicator2代表两个不同的指标，act func代表激活函数，最后通过multiply实现激活。

**其次是时序分域，时序分域是指对个股自身的数据序列进行一定的筛选再计算** 。在时序分域中，常见的是离散型分域的方法，也有连续型分域的方法，连续型分域的方法可以参考Alpha灵感中凸显因子的计算方式，本质上也是重新对时间序列数据进行重新加权，如 [[L2] 【Alpha灵感】凸显理论Salience Theory_20262909489431.md]([L2] 【Alpha灵感】凸显理论Salience Theory_20262909489431.md)

**而在离散型分域中，** 也是通过先建立筛选指标A，通过一些方式筛选出B中特定是数据点，再对这些数据点进行处理。

1. 指标A既可以是相关数据，也可以是另类数据。在Brain上，我们可以实现另类数据的时序切割，就比如特定的时间和时点。例如序列中最大值和最低值，某个datafield最大值出现的日期和最小值出现的日期，通过这样的方式做对比。

相关的运算符：ts_min_max_cps, ts_rank, ts_arg_max(A) – ts_arg_min(B)

1. 在研报中，我们也可以有时也会见到使用相关数据进行筛选的方法。比如 **开源金工所提出的“因子切割论”** ，他们通过成交笔数、单笔成交金额或者上下午动量等对其他量价数据进行分域建模，最后得出有效的因子。部分方法在Brain上可以实现，[1]中提出了优化早盘收益因子的方法，即绝对值将大于2%的收益率取相反数，在Brain平台上的简单实现可以是

```
new_rtn_series = abs(returns)>2%?-return:return;ts_sum(New_rtn_series, 20)
```

而其他方式暂时较难实现，期待后续新的运算符可以进行相应的操作。另外，在vector数据中，我们已经可以 **通过vec_filter进行一些简单的分域操作** ，比如筛选掉为nan的值。

1. 最后， **至于分域建立多因子模型** ，实际上与截面-离散型分域有异曲同工之妙。只是前文我们注重强调在域中的因子计算；而此处可以是先在当前时刻，通过某个标准对原始股票池进行分域，在不同的域中筛选表现较好的因子进行合成。
2. 动态情景Alpha模型是根据设定的标准，定期选出不同的Alpha进行多因子模型的合成，比如可以通过Alpha的Sharpe、IC等。动态情景Alpha模型还可以称作为因子择时。

***4. 分域中的常见问题***

分域确实是一种研究思路，但也有一些问题需要规避。

1. **分域指标难以确定。** 在阅读大量研究报告后，我发现分域指标没有大一统的确立方法，除了如行业、估值这一类如公司特征之外，很难发现有一种方法能明确识别有效的分域指标。大部分还是通过逻辑出发，进行Human Alpha的研究。以下是我遇到过的一些分域指标整理：

2.  **分域后，域中股票数量太小** 。比如属于统一subindustry的股票较少，因为子行业分的实在是太细了，可能某些子行业只有10-20只股票，这并不是我们想要的。所以在确定分域指标后，我们需要考虑域中的覆盖率。

3.   **如何通过多个指标进行分域？**

1. 有的时候我们想要考虑通过两个或更多指标同时进行分域，参考[3]，其中提到双重排序法。第一种是独立双重排序法，假设我们有两个分域指标X1和X2，它们分别独立地把股票划分成5组，它们两两取交集一共得到25个投资组合，在传统因子投资中这种投资组合用于检验因子是否有效，而在因子建模中，25个投资组合就是25个域，我们可以在25个域中进行差异化的操作。但这种方法容易遇到覆盖率不足的问题；
2. 第二种是条件双重排序法。它与独立双重排序最大的区别是按照给定的顺序先后使用两个变量对股票进行排序。仍然以X1和X2两个变量为例。假设先用X1排序 将全部股票划分成L1组。接下来，在以上每个组内，再用X2排序把属于该组内的股票进一步划分为L2个组，最终得到L1×L2个分组。从这个例子中不难看出，条件双重排序是考察当X1控制之后，变量X2对股票收益率的影响。条件双重分组关心的是当第一个变量被控制之后，第二个变量是否对解释收益率有增量信息。[3]这两种方法都可以用于因子计算，但在使用时需要区别其差异。

```
第一种实现：<group_opt>(<group_opt>({datafield}, Group1), Group2)第二种实现：<group_opt>({datafield}, group_cartesian_product(Group1, Group2))
```

***5. 分域建模的实操***

我在本部分提供一个简单模板，模板来自于[2]中通过遗传规划确立的运算符。

```
multiply({fndData}, sigmoid(zscore({pvData})))
```

这个模板很简单，我在KOR市场通过model109数据已经跑出了两个，具体步骤如下：

1. 我们先要考虑什么样的指标适合分域，我们从什么思路入手。我这边的一个思路是通过量价指标对基本面数据进行分域，逻辑如下：有些公司基本面反映很好，但是可能这些公司的行情已经被交易完了，这里面存在超买超卖，我通过量价指标在截面上是否异常（zscore是否够大）去反映这种提前交易或过度交易，对基本面指标进行重新赋权，赚取未被关注的利润。 **这里就是通过情绪指标去优化基本面数据**

```
fundData = group_extra(vec_avg(mdl109_m3r_mtn_spe_se), 1, group_cartesian_product(country, sector));pvData = group_extra(vec_avg(mdl109_news_sent_1m), 1, group_cartesian_product(country, sector));multiply(group_rank(fundData,industry), sigmoid(zscore(group_rank(winsorize(pvData),industry))))
```

```
group_rank(filter(sigmoid(if_else(greater(ts_zscore(news_sentiment, 30), 1),        ts_zscore(news_sentiment, 30),        0)),h="1 2 3 4",t="0.5"),industry)
```
