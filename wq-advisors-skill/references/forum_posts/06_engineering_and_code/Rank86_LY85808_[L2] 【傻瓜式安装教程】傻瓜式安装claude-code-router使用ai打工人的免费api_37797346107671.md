5.如果打开ai打工人提示一些报错的话，

--------------------------------------比方如下的黄字：-------------------------------

‼Auth conflict: Both a token (ANTHROPIC_AUTH_TOKEN) and an API key (ANTHROPIC_API_KEY) are set. This may lead to
  unexpected behavior.
    • Trying to use ANTHROPIC_AUTH_TOKEN? Unset the ANTHROPIC_API_KEY environment variable, or claude /logout then say
    "No" to the API key approval before login.
    • Trying to use ANTHROPIC_API_KEY? Unset the ANTHROPIC_AUTH_TOKEN environment variable.
  /model to try Opus 4.5

--------------------------------------------------------------------------------------------------------------

需要检查你的环境变量，命令如下：

使用windows+R

```
sysdm.cpl
```

这个命令选择高级-环境变量----检查环境变量是否出错，

ANTHROPIC_AUTH_TOKEN 
ANTHROPIC_API_KEY

这俩环境变量删掉然后去重做第四步就行

不懂的部分可以让ai帮你翻译一下。

如果有其他的api需要介入的话前面还有ccr 的ui界面，你在3的选项模板当中也给你集成好了很多别的类型的配置，这里的话不重复了。

基本上的话不需要你敲很多代码，反复调试即可。

很废话部分的原理介绍：

因为我们的ai打工人的话是基于claude-code开发的，目前只集成了这两种，这个路由工具的话，是将任何模型转换为anthropic格式的api，便于配置不同模型，发挥各自长处。
