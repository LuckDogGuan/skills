## 帖子正文

此工具可利用一个alpha的turnover数据，估算该alpha的交易费用、滑点损失等，再将原return与此费用/滑点损失相减，估算出费后真实的return。

这个计算方法的源起是一个在论坛/研究小组群里广泛流传的说法，即交alpha时最好确认return>turnover，但是有很多顾问并不知道这么做的原因何在。这两个数据量纲都不同，怎么可以进行比较呢？

其实这条经验的原理是这样的，这也是今天这个计算费后return工具的原理：

令年度总交易成本为C，单次交易成本为c，年度换手量为T，日换手率为t，则，

C = c*T*2（即单次交易成本*年度交易量，乘2是因为一次换手包括买入卖出，实际交易了2次）;

T = t*252 （252个交易日）;

从而费后return AC_return = return-C = return-2*c*T = return-2*252*c*t = return-504*c*t

然后我们假设c=20bp（万20），那么AC_return = return-1.008*t，发现了吗，这种情况下，只有在return>1.008*turnover时，费后return才是正的！否则这个策略就是不挣钱甚至亏钱的！而这正是上文那条经验法则背后的原理。

至于c的设置，可以根据不同市场的情况进行调节，实际上在美股这种高流动性市场里交易成本没那么高，大概5-10bp，用20bp假设算是个保守的策略，可能在交易费用之外还考虑了滑点等因素，因此设为20bp差不多。在ASI等市场可以再设置地高一些。
