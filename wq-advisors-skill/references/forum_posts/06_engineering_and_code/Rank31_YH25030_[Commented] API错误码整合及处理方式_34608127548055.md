## 帖子正文

相信大家的模拟回测过程中应该都遇到过各种错误码，并且这些错误起初都会对回测效率造成很大的打击，为了能够有更好且不间断的回测效率，就需要对各种错误码有一个正确的处理，现将本人整理的错误码解释及处理方式列举如下，如有不足之处还请批评指正，感谢各位！

1. 错误码——400
   解释：400错误码一般表示请求错误，多为请求参数不正确导致
   解决：遇到该错误码时检查自己的请求参数，确保请求参数正确
2. 错误码——401
   解释：401错误码为授权失败或者授权过期的意思
   解决：遇到该错误码时可再次发起授权请求进行API授权
3. 错误码——403
   解释：请求拒绝，多于提交alpha请求时出现，表示alpha不符合条件
   解决：遇到该错误码时剔除该alpha进行下一个alpha的提交
4. 错误码——404
   解释：404错误码意为请求资源不存在，在模拟回测过程中可能因网络问题或服务问题等原因请求失败，但在alpha提交过程中多为alpha数据问题导致无法提交（偶尔重试有可能通过）
   解决：在模拟回测过程中遇到该错误码时从post请求开始重试该模拟、在alpha提交过程中遇到该错误码时可尝试重新提交一遍，若再不通过则剔除该alpha进行下一个alpha提交
5. 错误码——429
   解释：该错误码为请求过于频繁或请求频率超限
   解决：遇到该错误码时优先提取header中的 Retry-After 值进行sleep等待，若该值为空时自定义请求缓冲时间，我一般设置的是30s，可自行调整
6. 错误码——500
   解释：该错误码表示服务端异常
   解决：重试请求即可
7. 错误码——502
   解释：该错误码一般表示网关异常
   解决：重试请求即可
8. 错误码——504
   解释：该错误码一般表示请求超时
   解决：重试请求即可

本人遇到的错误码基本为以上这些，如有不足还请补充，细节问题可细究讨论！
