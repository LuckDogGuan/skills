### 评论 #5 (作者: XY20037, 时间: 3个月前)

这篇是非常典型的 **薅 OpenAI 免费额度** 的实战羊毛帖，核心思路和风险给你整理清楚：

### 一、核心玩法（楼主整套逻辑）

1. **免费模型变动**
   - OpenAI 对免费号开放过  `gpt-5.3`  codex 额度，后来被撸多了，现在回落到  `gpt-5.2`
   - 目前可用： **Team 版试用 1 个月** ，可邀请小号，能用  `5.4`
2. **批量搞号 = 批量拿额度**
   - 工具：自动注册机（临时邮箱 / 域名邮箱批量注册）
   - 把很多小号的 API Key 整合成 **账号池**
3. **反代 / 网关聚合使用**
   - 用  `sub2api` 、 `CLIProxyAPI`  这类网关把多账号统一成一个 API 地址
   - 可以接到「AI 打工人」、OpenCode、Claude Code 等工具里
4. **配套工具**
   - CSS 项目：直接把 codex 集成到 Claude Code
   - 新版注册机：自动注册 + 自动导入账号池

### 二、楼主后续补充的重要变化

- 大量薅羊毛后，官方已限制，免费号现在只能用  **gpt-5.2**
- 新方案：走  **OpenAI Team 试用** （绑卡），邀请小号，可用更高版本模型

### 三、适合人群 & 风险提醒

- 适合： **预算有限、需要 GPT 批量优化 Alpha 表达式** 的 Gold/Master 新人
- 风险：
  - 账号容易被风控、封禁
  - 第三方反代项目有隐私、API Key 泄露风险
  - 官方政策随时变动，羊毛不可持续

整体就是给 **低成本 AI 写因子** 提供了一条临时路线，适合短期过渡用。
