## 帖子正文

作为刚入行近两个月的量化顾问，我从初期的摸索到如今稳定输出，踩过不少坑，也积累了些经验，今天就把这些干货分享给大家，希望能给伙伴们一些参考！

## 一、alpha 提交的 “血泪史”

刚开始提交 alpha 时，我完全摸不着头脑，有时能挖到，有时却一无所获，后来发现数据集是关键。像 Analyst 和 Fundamental 数据集，产出 alpha 相对容易些。

起初，我生怕提交操作影响 vf，每天只敢提交一个。但随着挖掘成果变多，不提交处理起来太麻烦，只好慢慢增加数量，从一天两个到三个。有次尝试提交四个，结果出问题了！check 任务全是秒结束，网页测试后发现当天提交四个后就无法再 check。所以现在我选择稳扎稳打，每天提交三个，或者卡点下午一点前提交最后一个，这样能避免踩雷。

## 二、效率提升经验总结

### （一）回测速度优化

最开始用培训学习的三阶原始代码，我就察觉到有优化空间，将单线程改成多线程后，效率确实有所提升，一天能回测两万多一点。后来在论坛发现 wqb 库，它用异步代码实现回测，但改动太大，我索性自己实现异步回测。虽然效率有提升，但因为平台有并发限制，效果没有预期那么好。所以建议大家别在回测速度优化上花费过多精力。

### （二）alpha 筛选技巧

回测完一阶，筛选优质 alpha 进入二阶很关键。我采用 set name 和 tag 的方式进行管理，name 格式为 “250420a_07407”，前面是日期，代表开始时间，若当天测试多个数据集，用 a、b、c 区分，后面是 index 顺序，足够满足日常使用，能保证 name 不重复；tag 方面，一阶 tag01，二阶 tag02，三阶 tag03 ，依靠这两个标识，能精准筛选回测过的 alpha，还能清晰区分阶段。

### （三）数据集选取策略

数据集选取时，Pyramid alpha distribution 是不错的参考，按照 genius 中数据集分布，逐个地区测试就好。

### （四）剪枝方法实践

原始代码的剪枝功能可用，但论坛里的相关性剪枝思路更优，我已将其接入代码。虽然目前在 news 数据集测试还没产出 alpha，但刚接入不久，期待后续能提高效率。

## 三、check submittion 的实用技巧

借鉴论坛大佬经验，我先做本地自相关性检测，检测通过后开始 check。每次 check 完一个，设置 sleep 600 秒，如果正常 check，下次 sleep 时间减少千分之 5，如果 check fail，下个 sleep 增加 200。这样能尽量贴近限制时间，提高效率，不过还得根据实际结果微调。看到论坛高手能根据指标筛选优质 alpha，做完相关性检测直接提交，无需 check，这也是我未来努力的方向。

## 四、模板的现状与规划

目前我还在用原始的三阶模板，但产出的 alpha 质量不太理想。想要获得优质 alpha，后续得去探索更多模板，这也是我接下来重点突破的方向。

入行这两个月，有挫折也有收获，希望这些经验能帮到大家。让我们一起朝着 GM 的目标冲刺，在量化顾问这条路上越走越顺！

以上内容涵盖了我这两个月的关键经验。你若觉得哪里需要补充、修改，或者有其他想法，欢迎随时和我说。
