## 帖子正文

成为平台的有条件顾问已经一个多月了，结合自己近期的观察和论坛前辈们的经验，在此分享一些小小心得，希望对刚加入平台的朋友们有所帮助。

1. 控制 fundamental datafields 的使用数量

平台对fundamental 类型的数据字段的使用有一定限制。如果在同一区域中使用的 alpha 数量超过 50 且其中 fundamental 字段的占比超过 30%，该区域将被封禁 fundamental 类型的字段，这不仅会限制数据使用，还会影响金字塔点亮率。而金字塔是平台对顾问评级的重要依据之一，因此建议合理控制 fundamental 字段的使用比例。

2.  Margin 指标

除了 Sharpe 和 Fitness，Margin 也是我特别关注的一个指标。Margin 太低意味着策略可能在考虑滑点、手续费等因素之后并不盈利，会影响 Value Factor 。但 Margin 也不能孤立地看，还需要结合 Turnover一起理解。

Margin 实际上可表示为 Return与 Turnover之比，即：Margin=Return/Turnover，

根据官方文档对margin，return和turnover的定义，

Return=Annualized PnL/(Book Size/2)=(PnL/Book Size)∗T，

Margin=PnL/Total Dollar Traded，

Turnover=Total Dollar Traded/Book Size，

将以上关系联立，可得：

Return=Margin∗Turnover∗T，等价地，Margin=Return/Turnover

由此可见，Margin 实际上是 Return 与 Turnover 的函数，数值变化受到两者共同影响。若 Turnover 极低，即使 PnL 有所增加，由于分母极小，仍可能导致 Margin 被异常放大，产生“虚高”的现象。

举个例子：

某策略一年内只交易了 $10,000，但获得 $500 净利润，Margin = 5%。

若 Book Size 为 $20,000,000，则 Turnover = 0.0005，最终 Return 仅为 0.25%。

因此在筛选 alpha 时，应综合关注 Margin 和 Turnover，避免误判。

3. 把握 PPAC 比赛的机会

PPAC 比赛是新人快速积累金字塔数、实现每日alpha提交最大化的好机会，门槛较低，Sharpe 要求低，Prod Correlation 无需通过。有利于保证每日提交足够数量的 signal。不过也要注意，Prod Corr 太高会影响 base payment，alpha 质量过差会影响 Value Factor。因此建议权衡质量与数量。

4. 多样化使用数据字段

从平台的评级指标来看，如金字塔和 Fields Used，数据使用的多样化至关重要：

建议尽量使用不同地区、不同类别（Category）的数据；面对新的数据集，可以优先从 coverage 高、universe 大的字段入手；每个数据集都附带 resource，建议都点进去看一看，里面通常会有字段介绍、策略思路等，非常适合新手快速入门；

特别注意：在不同的地区、universe 和 delay 下，字段名字相同，如 "split" 或 "dividend"，它们被视作同一个的字段，在 Fields Used 中只算作一个。
