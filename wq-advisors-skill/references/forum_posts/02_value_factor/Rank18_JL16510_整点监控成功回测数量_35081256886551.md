## 帖子正文

每次平台更新vf或者其他，回测就会受影响，一般情况速度变慢，但严重可能导致程序运行停止或停滞。最近更新vf，某天晚上程序运行停滞，一晚上没回测，于是决定实时监控成功回测数量，主要为了及时获知回测是否正常运行。

首先选择wxpusher作为消息推送工具，具体实现参照@LL62621的帖文，连接如下 [[L2] 代码微信通知_28941872349975.md]([L2] 代码微信通知_28941872349975.md@CT98586%E7%9A%84%E7%AE%80%E6%98%93%E7%89%88%E6%9C%AC%E3%80%82) 。但由于微信官方限制，其微信公众号暂停发送消息，可用app替代。这部分代码使用评论区@CT98586的简易版本。

其次，更改get_alpha函数代码（也加入课代表发送请求重试部分的代码），使其返回特定时间段内的成功回测数量。

最后，利用任务计划程序定时运行py脚本（window系统自带）。注意选择创建任务并设置单选每天，勾选重复任务间隔并选择1小时（截图传不上）。

全部代码如下

def get_alphas_num(start_date, end_date):
    s = login()

retries = 0

# 可选：检查内容是否为空

            elif response.json().get("results", 0) == 0:
                print("有问题")
                retries += 1
                # print(response.json())

except requests.exceptions.ReadTimeout:

            retries += 1

except requests.exceptions.RequestException as e:
