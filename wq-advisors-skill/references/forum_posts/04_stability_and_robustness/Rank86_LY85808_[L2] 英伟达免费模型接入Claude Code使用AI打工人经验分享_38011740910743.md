最近 Nvidia（没错，就是那家做显卡的英伟达）上线了两个免费的国产模型，分别是  **glm-4.7**  和  **minimax M2.1** 。

众所周知，这两个模型都相当不错。最重要的是，英伟达官方提供了直接调用的 API， **目前的限制是 40 RPM（每分钟 40 次请求）** ——约等于无限制。

既然有这么好的免费资源，我们是否可以把它应用到  **Claude Code**  上呢？毕竟 Claude 官方封号严重，市面上的中转服务也不够稳定。

答案是可以的。通过简单的配置，就可以愉快地使用 Claude Code。以下是详细教程。

# 第一步：获取 Nvidia 免费 Key

**1. 进入注册页面** 访问： [https://build.nvidia.com/](https://build.nvidia.com/)

根据指引使用邮箱注册。验证邮件可能会被误判为垃圾邮件，如果没有收到，请检查一下垃圾箱。

**2. 验证身份** 注册成功后，随意创建一个英伟达云账号（名称随意填）。系统会要求验证手机号。

点击右上角的“Verify”进行验证。 **亲测支持 +86 大陆手机号** 。

小贴士：如果无法选择地区，请检查网络连接环境。

**3. 生成 API Key** 验证完成后，点击页面右上角头像，选择  **API Keys**  进入管理页面。

点击生成 Key。

⚠️  **重要提示** ：Key 只会完整显示一次，请务必第一时间复制并妥善保存。

# 第二步：安装中转工具 claude-code-router

Nvidia 提供的 Key 格式与 Claude Code 原生不兼容，这就需要一个“翻译官”。推荐使用 GitHub 上的开源项目 claude-code-router。

[https://github.com/musistudio/claude-code-router](https://github.com/musistudio/claude-code-router)

它可以将不同厂商的 API 伪装成 Claude Code 能够识别的格式。

**安装前提**

- 已安装 Claude Code。
- 已安装 Node.js ( v18 以上版本)。

**安装命令：** 打开终端，输入以下命令即可完成安装和启动：

- 全局安装工具

```
npm install -g claude-code-router

```

- 启动服务

```
ccr start

```

- 进入配置页面

```
ccr ui

```

浏览器会自动打开配置界面。注意保持这个终端窗口不要关闭。

# 第三步：配置核心参数（关键）

这是最容易出错的一步，需仔细核对以下信息。

在网页 UI 中点击**添加供应商 (Add Provider)**，按照下方卡片的内容进行填写：

# 配置信息

**1. API 地址 (Base URL)** 请直接复制下方地址，不要多空格：

```
https://integrate.api.nvidia.com/v1/chat/completions

```

**2. API Key**  填入在第一步中保存的 nvapi-xxx 开头的密钥。

**3. 供应商转换器 (Transformer)**  选择openrouter

**4. 模型名称 (Model Name)**  请分别复制以下模型 ID 填入并回车（建议两个都添加）：

- GLM-4.7 模型：

```
z-ai/glm4.7

```

- Minimax M2.1 模型：

```
minimaxai/minimax-m2.1

```

# 第四步：开始白嫖

回到你的终端，进入项目文件夹，不再使用 claude 命令，而是使用以下命令启动：

### 评论 #10 (作者: YP74336, 时间: 4 months ago)

楼上提到ccr start 执行后卡住，及ccr ui提示错误的，原因是贴主提供的安装命令( `npm install -g claude-code-router` )可能已经失效，需要先卸载，然后重新用正确的命令安装ccr。

1.先用下面的命令卸载之前安装的ccr

```
npm uninstall -g claude-code-router
```

2.然后用GitHub中最新的命令安装，下面这个命令

```
npm install -g @musistudio/claude-code-router
```

`安装后就可以打开ccr ui了`
