### 评论 #18 (作者: WQ47560, 时间: 1个月前)

反馈内容：修复 Windows 环境下 Python 路径包含空格导致无法启动脚本的问题在webui中点击问题描述： 在 Windows 系统中，如果 Python 安装在包含空格的路径下（例如 C:\Users\Name\App Data... ），原代码使用字符串拼接方式调用 cmd /k 时， cmd.exe 会错误地剥离引号，导致报错： 'C:...\python.exe" "script.py' 不是内部或外部命令 。修复方案： 建议将 subprocess.Popen 的调用方式从 字符串形式 改为 列表 (List) 形式 。这样 Python 会调用 Windows 的 CreateProcess API 自动处理参数转义，无需手动处理复杂的 cmd 引号逻辑。修改代码如下：修改 run_simulator 函数（约 2179 行）：# 修改前：subprocess.Popen(f'cmd /k "{APP_PYTHON}" "simulator_wqb.py"',cwd=simulator_dir,env=APP_ENV,creationflags=subprocess.CREATE_NEW_CONSOLE)​# 修改后（推荐）：subprocess.Popen(["cmd", "/k", APP_PYTHON, "simulator_wqb.py"],cwd=simulator_dir,env=APP_ENV,creationflags=subprocess.CREATE_NEW_CONSOLE)修改 open_submitter 函数（约 2284 行）：# 修改前：subprocess.Popen(f'cmd /k "{APP_PYTHON}" "alpha_submitter.py"',cwd=simulator_dir,env={**APP_ENV, **submitter_env},creationflags=subprocess.CREATE_NEW_CONSOLE)​# 修改后（推荐）：subprocess.Popen(["cmd", "/k", APP_PYTHON, "alpha_submitter.py"],cwd=simulator_dir,env={**APP_ENV, **submitter_env},creationflags=subprocess.CREATE_NEW_CONSOLE)修改 open_hk_simulator 函数（约 2370 行）：# 修改前：subprocess.Popen(f'cmd /k "{APP_PYTHON}" "autosimulator.py"',cwd=hk_simulator_dir,env=APP_ENV,creationflags=subprocess.CREATE_NEW_CONSOLE)​# 修改后（推荐）：subprocess.Popen(["cmd", "/k", APP_PYTHON, "autosimulator.py"],cwd=hk_simulator_dir,env=APP_ENV,creationflags=subprocess.CREATE_NEW_CONSOLE)修改 open_transformer 函数（约 2449 行）：# 修改前：subprocess.Popen(f'cmd /k "{APP_PYTHON}" "Transformer.py"',cwd=transformer_dir,env=APP_ENV,creationflags=subprocess.CREATE_NEW_CONSOLE)​# 修改后（推荐）：subprocess.Popen(["cmd", "/k", APP_PYTHON, "Transformer.py"],cwd=transformer_dir,env=APP_ENV,creationflags=subprocess.CREATE_NEW_CONSOLE)修改原理： 当 subprocess.Popen 接收一个列表时，它会自动为包含空格的每个元素添加必要的转义，确保在 Windows 终端中作为一个完整的路径被识别。这解决了 cmd.exe /k 在处理多个带引号参数时的解析歧义。
