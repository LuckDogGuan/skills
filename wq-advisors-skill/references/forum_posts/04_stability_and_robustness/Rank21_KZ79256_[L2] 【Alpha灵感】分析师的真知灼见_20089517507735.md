## 帖子正文

研报标题：《剥离分析师预期调整中的动量效应与真知灼见因子构建》及《超预期陷阱与估值动态及“预期惯性”因子构建》

作者：曹春晓，方正金融工程团队

概述：首篇研报致力于解决传统一致预期因子失效的问题。在考虑每个分析师不同见解的基础上，剥离研报发布前后的动量效应；第二篇致力于解决首篇中”真知灼见“因子在观察期仍然失效的问题。

**Alpha Idea:**

将动量效应从分析师预期中剥离，是因为研报发出前，部分股票的价格可能会受到知情者交易问题，从而分析师预期被迫提前反应在股价上，误导真正投资者的交易，而本文提出的PAFR因子，就是希望剥离到这些被操纵的因子，从而反应真正有价值的股票。

**主要数据字段：** oth41_s_west_netprofit_fy1_3m

**主要运算符：** vec_count, tail, regression_neut

**Universe:** CHN TOP3000和TOP2000；需要额外注意的是，研报中

**1. 因子构建**

**1.1 浅谈CAFR**

首先考虑一个比较常见的一致预期因子构建。一致预期数据对于个股来说是vector，一个vector上储存了不同分析师的见解。通常情况我们直接对数据取平均值，此时直接当作因子或计算变化率都可以。以下展示了一个以一致预期利润数据构建的CAFR绩效。

**1.2 AFR因子**

该因子与CAFR的差异在于，它先考虑了每个分析师的预期变化率，再考虑个股一致预期层面，具体来说

1. 先计算每个分析师，FY2预期净利润的90天变化率，由于未发现有FY2字段的数据，我们此处用oth41_s_west_netprofit_fy1_3m替代，这个字段为FY1预期净利润的90天变化率；
2. 对于每一个分析师的预期变化，需要做一些去异常值处理，研报将值限制在（-0.25，0.25），我们可以用两次tail运算符实现这个功能。需要说明的是，研报并没有说0.25这个关键点是如何定义的，只是说为了防止数据过高影响策略性能，当然，笔者也尝试过使用winsorize取极值，发现绩效近似，因此后续依然采用tail-tail
3. 去极值后，进行vec_avg便可以获得AFR；
4. 对于股票池，我们需要手动筛选。如果在过去90天内，对于某股票的报告少于3份，予以剔除；
5. 最后，再解释一下前文中为什么要对TOP3000和TOP2000都要进行尝试的原因。在TOP3000中回测，会出现某一个节点，权重非常大的情况，这有可能是因为分析师更喜欢分析成长性的股票，而很大一部分成长性股票的市值较小，如果我们不做中性化处理或其他合理的处理，那么很有可能效果好而无法通过检测，除此之外，相对大市值的TOP2000，该情况几乎不会发生。

**1.3 PAFR因子**

此处我们构建”真知灼见“(RAFR)因子。我们先观察时间轴

观察这个时间轴，可以发现分析师alpha在实际使用前，就有可能会有损耗。因为研报是月频调仓，所以在调仓实际使用时，分析师的预期已经被反应了一部分，这个问题在Brain平台影响不大，但分析师数据还有一个不可忽视的问题：知情交易对alpha的影响。

在分析师发布研报前，很有可能因为知情交易的问题，导致一部分预期已然反应在股价上，而真正发布研报后，投资者反而被误导。但是知情交易者的投资广度并不是很大，这会使部分股票的预期有大幅修正、而短期内股价变化较为平稳。假设一个只有A、B两只股票的市场，分析师觉得A股票的净利润可能会上升40%，B股票会上升20%，由于知情交易只知道B股票的内幕并在B股票上提前充分交易，在研报发出后，A股票可能仍然“安全“。因此，构建预期修正幅度大、涨跌幅小的组合是理想的。具体来说，PAFR的构建方法如下：

1. 计算个股vector上每一格数据相较于上一期数据发生变化的时间间隔，可以用days_from_last_change实现。假设现在是t日，t-30日和t-60日分别发布了研报，通过这一个运算符，我们可以分别得到day=30和60，再将这个数据放入ts_mean(returns,day)，就可以得到时间轴上的两段动量。但似乎Brain目前不支持，不知道各位老师或同学有没有实现方法；
2. 因此，我们这里构建一些代理变量。首先对于Δt3，橙色箭头里的短期动量：我们简单将其视为近一周的动量，它刻画了研报发出后的市场的消化程度，这就是我们想要寻找的涨跌幅较小股票的指标，我们将其记为mom1；
3. 对于Δt2，灰色箭头里的长期动量：我们简单定义为ts_delay(ts_mean(returns,20),20)，两次研报发出期的间隔为20天，再delay掉，说明是过去一段时间的动量，让它和短期动量有所区分，记为mom2;
4. 我们要剥离上述两种动量，进行截面回归。对于mom1，剥离掉它，利于我们观察到研报发出后，是哪些股票充分消化了研报的预期，哪些没有。没有消化的股票，残差值会较大；
5. 对于mom2，我们希望剥离掉研报发出前被内幕交易掉的涨跌幅，如果股价在发出前没有受到内幕交易影响，可能股价更为平稳，更为平稳的股票在回归后，残差值大。最后得到momentum风格的因子；
6. 一些comment：在研报中的月频调仓下，该因子构建方式是比较合理的，即考虑到研报前期的动量，也考虑到发出后的动量，相对来讲，调仓日与研报公布日重合次数可能比较少；而日频调仓下，很有可能与公布日重合，这会导致短期动量mom1的回归看上去不太有用。（如果尊重回测结果的话，那还是两次回归的效果较好）
7. 在构建完上述框架后，文中提到的动量为超额动量。即个股returns减去市场截面returns均值，再求20日均值。因此，需要将前序步骤中的returns部分替换为excess return

**1.4 预期惯性因子**

这个因子出自于后一篇，原因是研究员在样本外跟踪PAFR时，发现2023年内该因子全面失效，所以做了优化。逻辑为：投资者通常会将实际兑现的业绩增速与之前的预期进行比较，进而对股票的估值产生正面或负面的反馈，我们认为这种现象源于投资者的思维惯性。因此我们基于这样现象出发，提出“预期惯性”因子，以更好地解释和利用市场对业绩预期变化的反应模式和估值的反馈。具体构建方式为将PAFR做时序zscore，从而模拟投资者的比较估值行为，可以用ts_score实现，d取252，以下为测试结果

### 评论 #2 (作者: YW93864, 时间: 2 years ago)

[TN48752](/hc/en-us/profiles/13714359745431-TN48752)  您好，感谢您的认可，我很乐意展示出该alpha的表达式。

1. 首先限定股票池，回望过去90天，观察个股研报发布数量是否大于三份，如果没有则不交易。（分析师发研报一般不会遵守交易日历，因此采用自然月天数90），此处用90天（3m)的数据作为例子

```
ts_sum(vec_count(oth41_s_west_eps_ftm_chg_3m),90)<=2?nan:afr1
```

2. 根据研报，对alpha进行异常值处理，此处因为限定了数值在正负0.25之间，我们使用两次tail完成（需要注意的是，此处取1000没有任何意义，只是为了设置一个极大值）

```
modified1 = tail(oth41_s_west_eps_ftm_chg_3m,lower=0.25,upper=1000,newval=0.25)modified2 = tail(modified1,lower=-1000,upper=-0.25,newval=-0.25)afr1 = vec_avg(modified2)
```

3. 分别定义长期和短期超额动量。短期超额动量是衡量研报发出日附近的短期内（比如一周），是否有内幕交易把分析师的预期提前交易掉；

```
short_term_excess_return = ts_mean(returns-group_mean(returns,1,market),5)
```

长期超额动量是衡量研报发出日长期内（比如研报发出一个月前），是否有内幕交易把分析师的预期提前交易掉。

```
long_term_excess_return = ts_delay(ts_mean(returns-group_mean(returns,1,market),20),20)
```

需要注意的是，1. 此处采用交易日衡量动量；2. 长期超额动量需要有一定delay，请品味前文提供的箭头。

4. 剥离掉动量，使用两次regression_neut，这个是取残差（regression_proj是取预测值）

```
parf = regression_neut(regression_neut(modified2,short_term),long_term)
```

由于是线性模型，回归的先后顺序理论上不会对结果产生影响，以上为构建的主要逻辑，请对我提供的表达式进行适当调整便于回测。

逻辑为： **如果有内幕交易，那么投资者最后看到的分析师预期将不具备投资指导作用，所以需要剥离** 。比如，即使一家公司预期利润率变化是+100%，然而有内幕交易者提前将这部分利润交易掉了，那么这个100%反而是误导性的，-100%的情况类似，那如何处理？研报使用截面回归剥离取残差，那么预期得到的数值是：如果公司预期很好/很差+内幕交易，剥离后的数值应该较小；如果公司预期很好/很差+无内幕交易，剥离后的数值依旧很大；至于中性化后为正还是为负，取决于预期本身。

**综上因子为momentum。另外，分析师其实也有一些基本面属性，基本面常用的方法是行业内进行比较，因此推荐sector和industry的中性化**

5. 对于第二篇研报的优化，分析师只是简单取了ts_zscore(parf,252)

PS：我尝试了很多分析师相关的数据集，该逻辑依然可以创造出很多足以提交的alpha。当然因为分析师数据在Brain上非常非常丰富，我还未能穷尽，如果您有进一步的成果和改进，欢迎交流~
