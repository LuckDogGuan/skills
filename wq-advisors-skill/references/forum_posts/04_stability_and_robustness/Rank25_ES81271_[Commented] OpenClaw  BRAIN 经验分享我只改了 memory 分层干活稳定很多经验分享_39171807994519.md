## 帖子正文

最近折腾了一圈原生和二开的 OpenClaw，最后一个很直观的感受是：BRAIN skill 真正有用，不在于它“多会想”，而在于它“会不会把东西记对地方”。

一开始我也以为装完 BRAIN、接好模型、能跑起来就算结束了，结果实际干活时经常出现三个问题：

1. 刚聊顺一点，开个新 session 又像失忆。

2. 项目状态、用户偏好、当天临时结论全堆在一起，越记越乱。

3. 同一个任务隔一天再做，明明文件还在，关键上下文却总是抓不准。

后来我没有继续堆 skill，而是先把 workspace 里的几份记忆文件重新分层，效果立刻稳定很多。

我现在的最小配置

- `SOUL.md`：只写人格、边界、做事风格，不写项目流水账。

- `USER.md`：只写用户偏好、长期习惯、常见约束。

- `MEMORY.md`：只放长期有效的信息，比如项目现状、稳定规则、固定环境说明。

- `memory/YYYY-MM-DD.md`：只记当天结论，不记大段聊天过程。

- `AGENTS.md`：规定每次 session 先读什么、遇到什么情况要写回哪里。

我现在在 `AGENTS.md` 里固定了一段很土但很好用的规则：

```md

## Every Session

Before doing anything else:

1. Read `SOUL.md`

2. Read `USER.md`

3. Read today's and yesterday's `memory/YYYY-MM-DD.md`

4. If this is a main/private session, also read `MEMORY.md`

## Memory Rules

- Store stable facts in `MEMORY.md`

- Store daily conclusions in `memory/YYYY-MM-DD.md`

- Record conclusions, not full chat transcripts

- When a project status changes, update the related memory file

```

关键点不是“多记”，是“少而准”

我后来专门改掉了一个坏习惯：不让它把操作过程整段写进记忆。

比如部署失败三次、试了五个命令，这些不该原样塞进长期记忆；真正该留下来的，是最后成功方案是什么、失败雷点是什么、下次别再踩什么坑。BRAIN 检索再强，底层如果全是流水账，它找回来的也只会是一堆噪音。

所以我现在的原则就是一句话：记结论，不记废话；记变化，不记表演。

这样配完之后，干活明显稳了

1. 新 session 明显没那么“失忆”了。

2. BRAIN 检索历史时，更容易命中真正有用的结论。

3. `SOUL.md` 和 memory 不再互相污染，人设是人设，项目是项目。

4. 做重复活时，OpenClaw 更像“接着上次干”，而不是“重新猜一遍”。

如果你现在也在用原生或者二开的 OpenClaw，已经把 BRAIN skill 装上了，但还是觉得它干活不稳、容易忘、越用越乱，我真心建议先别急着堆更多 skill，先把记忆层级理顺。这一步做好了，后面的 Soul、定时任务、工具调用都会顺很多。
